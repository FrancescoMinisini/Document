\subsection{Compared methods}
Within this common framework, four optimization and comparison schemes are considered. 
\begin{itemize}
    \item The first is a direct gradient-based baseline, implemented through the Adam optimizer, which searches directly over the filtered control trajectory for a fixed gate horizon. 
    \item The second is a policy-gradient reinforcement-learning controller trained in the nominal environment, that is, without stochastic perturbations applied during training. 
    \item The third is the noise-optimized TRPO controller, which is trained in a stochastic environment where Gaussian perturbations are injected into the control amplitudes and into the anharmonicity-related parameter entering the Hamiltonian. 
    \item The fourth comparison is not a pulse-level optimizer but a circuit-level reference, namely the optimal gate-synthesis baseline used to benchmark runtime on the family \(N(\alpha,\alpha,\gamma)\). 
\end{itemize}
These four methods play different methodological roles. The Adam baseline tests whether a standard first-order optimization of the raw control variables is sufficient to obtain high-quality pulses. The nominal TRPO controller tests whether reinforcement learning is already competitive in the idealized deterministic setting. The noise-optimized TRPO controller tests whether robustness can be learned directly from stochastic training. Finally, the gate-synthesis baseline tests whether direct analog control offers a runtime advantage over a discrete compiled implementation of the same target gate. Taken together, these methods define a comparison protocol that is not centered on a single optimizer, but on a set of scientifically distinct questions concerning accuracy, robustness, and hardware efficiency.

\subsection{Metrics}
The performance of each control method is evaluated through a set of complementary metrics designed to probe different aspects of the control problem. At the most basic level, the implemented gate is assessed through its fidelity with respect to the target unitary. For a final physical propagator \(U(T)\), projected onto the computational subspace, the gate fidelity is computed as
\begin{equation}
F_{\mathrm{gate}}
=
\frac{1}{16}
\left|
\mathrm{Tr}\!\left(
U_{\mathrm{proj}}^{\dagger}(T)\,U_{\mathrm{target}}
\right)
\right|^2,
\label{eq:validation_gate_fidelity}
\end{equation}
where \(U_{\mathrm{target}}=N(\alpha,\alpha,\gamma)\). This quantity is reported together with the total UFO cost, which combines fidelity, leakage, boundary penalties, and runtime into a single scalar objective. Since the framework is explicitly multilevel, nominal performance is not characterized by fidelity alone. For every optimized control plan, the simulation also returns the TSWT-based leakage estimate \(L_{\mathrm{tot}}\), its decomposition into boundary and integral contributions, the boundary penalty term, and the final runtime in nanoseconds. These quantities allow one to distinguish between a control that is merely accurate on the logical subspace and one that is also physically well behaved in the full truncated Hilbert space. In addition to nominal performance, robustness is evaluated statistically under sampled noisy control realizations. Given a control plan \(\tilde{\bm{c}}\) and a Gaussian noise level \(\sigma_{\mathrm{noise}}\), the key robustness diagnostics are the average fidelity
\begin{equation}
\overline{F}
=
\mathbb{E}_{\delta\tilde{\bm{c}}}
\Bigl[
F\bigl(U(\tilde{\bm{c}}+\delta\tilde{\bm{c}})\bigr)
\Bigr],
\label{eq:validation_average_fidelity}
\end{equation}
the corresponding average gate fidelity, and the fidelity variance
\begin{equation}
\mathrm{Var}(F)
=
\mathbb{E}_{\delta\tilde{\bm{c}}}
\left[
\bigl(F-\overline{F}\bigr)^2
\right].
\label{eq:validation_fidelity_variance}
\end{equation}
These metrics are essential because they distinguish mean robustness from shot-to-shot stability: a control scheme with high average fidelity but large variance remains experimentally fragile. Finally, for the runtime study, the relevant metric is the total control duration required to realize a target gate, compared against the optimal discrete gate-synthesis baseline. The resulting evaluation protocol is therefore explicitly multidimensional: it measures logical accuracy, leakage suppression, robustness under stochastic perturbations, and hardware efficiency rather than relying on a single scalar score.

\subsection{Evaluation protocol}
All experiments follow a common procedural logic. For a given target gate \(N(\alpha,\alpha,\gamma)\), a control method is first trained or optimized under its own learning rule until either a prescribed iteration budget is exhausted or a termination criterion on the UFO cost is satisfied. The resulting best control plan is then stored and re-evaluated in a deterministic nominal setting in order to extract its final cost, fidelity, leakage, and runtime under identical physical assumptions. For robustness studies, the same saved control plan is subsequently subjected to repeated noisy rollouts, in which Gaussian perturbations are independently sampled and injected into the control channels and into the anharmonicity-related parameter of the Hamiltonian. This Monte Carlo evaluation yields the empirical estimates entering Eqs.~\eqref{eq:validation_average_fidelity} and \eqref{eq:validation_fidelity_variance}. In the runtime study, by contrast, the focus is not on statistical robustness but on how the optimized gate duration varies across the target family, especially along the subfamily \(N(\alpha,\alpha,\pi/2)\), where analog control can be meaningfully compared with optimal gate synthesis. The same timestep, control bounds, bandwidth assumptions, and physical truncation are retained throughout, so that differences between methods can be attributed to the optimization strategy rather than to inconsistencies in the simulation model. This common evaluation protocol is particularly important for the interpretation of the results: it ensures that the subsequent comparisons are methodologically fair, while also making the distinction between nominal accuracy, noisy robustness, and runtime efficiency completely explicit.




\section{Parameter-choice study}
\label{sec:parameter_choice_study}

The empirical performance of the present control framework depends not only on the underlying physical model and on the definition of the UFO objective, but also on a number of methodological choices that must be fixed before a meaningful comparison can be carried out. This is especially true in the reinforcement-learning setting considered in this thesis, where optimization quality is affected simultaneously by the training configuration, by the expressive capacity of the policy and value networks, and by the scalarization adopted in the control cost. For this reason, parameter selection should not be treated as a minor implementation detail. On the contrary, it constitutes an explicit part of the experimental methodology, since different choices may alter convergence stability, final gate quality, robustness under noise, and even the apparent relative performance of the compared optimizers.

A further reason for discussing parameter choice carefully is that the reference framework of Niu \emph{et al.} fixes several ingredients rather explicitly. In particular, the original study adopts a continuous-action TRPO agent with two fully connected neural networks of hidden sizes \(64\), \(32\), and \(32\), updates the policy after batches of \(20000\) episodes, injects Gaussian amplitude fluctuations with a standard deviation of \(1\,\mathrm{MHz}\) during stochastic training, and uses the same UFO weight configuration \(\chi=\beta=10\), \(\mu=0.2\), \(\kappa=0.1\) across the target gates considered. The same paper also evaluates robustness on the representative gate \(N(2.2,2.2,\pi/2)\), which therefore provides a natural reference target for model-selection purposes in the present reproduction. These design choices are important because they define the baseline notion of a \emph{fair comparison} with the original framework: if the thesis changes some of them, this should be done deliberately and justified explicitly, rather than implicitly through undocumented implementation drift. :contentReference[oaicite:0]{index=0} :contentReference[oaicite:1]{index=1} :contentReference[oaicite:2]{index=2} :contentReference[oaicite:3]{index=3}

At the same time, the goal of this section is not to claim a globally exhaustive hyperparameter optimization over the full design space. Such a program would be computationally prohibitive and would add little scientific value to the main questions of the thesis. The objective is instead more focused: to identify a parameter regime that is stable, reproducible, physically sensible, and sufficiently well aligned with the reference paper to support a credible reproduction study. In other words, the purpose of the parameter-choice stage is to reduce arbitrariness, not to optimize every numerical setting in isolation. This distinction is methodologically important. A thesis-level validation study should not depend on a hidden collection of ad hoc choices tuned separately for each experiment; it should rely on a small and transparent set of decisions that are fixed once and then used consistently throughout the empirical chapter.

For clarity, the parameter-selection analysis is therefore decomposed into three separate studies. The first concerns the \emph{training hyperparameters}, such as the timestep, maximum horizon, trust-region radius, exploration scale, batch size, and termination criterion. These parameters primarily affect optimization stability, sample efficiency, and computational cost. The second concerns the \emph{neural-network architecture} used for the policy and value functions. This is treated separately because the expressive capacity of the actor--critic representation is not simply another training detail: it may materially influence the quality of the learned control policy and therefore must be distinguished from purely algorithmic settings. The third concerns the \emph{UFO cost-function weights}, which play an even more special role. Unlike ordinary optimizer hyperparameters, these coefficients are part of the physical definition of the control objective itself, since they determine how gate infidelity, leakage, boundary regularity, and runtime are balanced against one another. For this reason, architecture choice and cost-function choice should not be conflated with the tuning of TRPO-specific parameters.

This separation is also useful from the viewpoint of interpretation. If network size and UFO weights were tuned simultaneously together with training hyperparameters, any later performance difference would become harder to attribute: an observed improvement could arise from a better optimizer configuration, from increased representational capacity, or from a different physical scalarization of the control problem. By isolating the three studies, the empirical discussion becomes much cleaner. The reader can first understand which choices are made for numerical stability, then which are made for representational adequacy, and finally which affect the physical tradeoff structure encoded in the reward. This is particularly important in the present thesis, because one of its goals is not only to reproduce the original framework, but also to clarify which parts of its performance are tied to the learning algorithm and which depend more directly on the structure of the cost function.

The remainder of this section is organized accordingly. Section~\ref{subsec:parameter_selection_tradeoffs} discusses the training hyperparameters and the practical trade-offs that motivate their final selection. The following subsection is devoted to the sweep over policy and value network sizes, with the aim of determining whether the paper architecture is already sufficient for the present implementation or whether a different capacity yields a more stable control regime. After that, a dedicated sensitivity study of the UFO coefficients is introduced, in order to separate the role of the physical objective from that of the optimizer itself. Finally, the section closes by fixing one final parameter configuration for the remainder of the chapter. Unless explicitly stated otherwise, all later experiments will be performed under that single configuration, so that the subsequent comparisons are not biased by experiment-specific retuning.

In this way, the parameter-choice study serves a double purpose. On the one hand, it provides the methodological foundation needed to interpret the numerical results that follow. On the other hand, it establishes the conditions under which the present work can be compared fairly with the original study of Niu \emph{et al.}, whose reported framework combines a stochastic TRPO environment, a fixed actor--critic architecture, a representative robustness benchmark on \(N(2.2,2.2,\pi/2)\), and a uniform UFO scalarization across the tested gate family. The parameter-selection stage is therefore not ancillary to the empirical chapter, but the step that makes its later claims scientifically interpretable.

\subsection{Training hyperparameters and practical trade-offs}
\label{subsec:parameter_selection_tradeoffs}

A first layer of parameter selection concerns the training hyperparameters of the reinforcement-learning pipeline. These choices do not define the physical control objective itself, but they strongly affect optimization stability, sample efficiency, and computational cost. For this reason, they should be discussed separately from both the neural-network architecture and the UFO cost-function weights, which will be treated in the following subsections. The purpose of the present subsection is therefore not to perform an exhaustive hyperparameter optimization, but to identify a training regime that is sufficiently stable, reproducible, and well aligned with the reference framework to support the experiments reported later in this chapter.

The training-parameter study is anchored to a representative gate from the target family \(N(\alpha,\alpha,\gamma)\), namely
\begin{equation}
U_{\mathrm{rep}} = N(2.2,2.2,\pi/2).
\label{eq:representative_gate_training_study}
\end{equation}
This choice is natural for two reasons. First, it provides a nontrivial two-qubit target that is sufficiently far from the identity to make the control problem genuinely informative. Second, it matches the representative robustness benchmark used in the original paper, and therefore offers a convenient reference point for tuning the implementation while preserving comparability with the published framework \cite{Niu2019}. In this way, the parameter-selection stage is not performed on an arbitrary auxiliary task, but on a gate that is already central to the main empirical narrative of the thesis.

Within this restricted setting, the main training parameters of interest are the timestep \(\Delta t\), the maximum admissible gate duration \(T_{\max}\), the trust-region radius \texttt{max\_kl}, the initial exploration scale \texttt{init\_log\_std}, the batch size measured in episodes per policy update, the number of training iterations, and the terminal-cost threshold used for early stopping. These quantities affect different aspects of the optimization. The timestep controls the temporal resolution of the pulse and therefore the expressivity of the sequential decision problem. The maximum duration defines the horizon over which the policy is allowed to construct the gate. The trust-region radius determines how aggressively the policy can move in one TRPO update, while the initial log-standard deviation controls the width of the Gaussian policy and therefore the amount of exploration at the beginning of training. Finally, the batch size and iteration budget determine the statistical reliability of the policy-gradient estimate and the total wall-time cost of the experiment.

The timestep and maximum horizon involve the clearest physical--computational trade-off. A smaller timestep gives the policy access to a finer control mesh and therefore to a richer set of pulse shapes, which is desirable in a multilevel system where the balance between fidelity, leakage suppression, and runtime may depend sensitively on pulse modulation. However, this also increases the episode length and makes the optimization problem higher-dimensional and more expensive. Similarly, a larger value of \(T_{\max}\) enlarges the space of reachable solutions, but it also makes exploration more difficult and weakens the hardware-efficiency motivation of the analog-control framework. In the present implementation, the default choice for the main runs is
\begin{equation}
\Delta t = 2\,\mathrm{ns},
\qquad
T_{\max}=180\,\mathrm{ns},
\label{eq:default_time_discretization_training}
\end{equation}
which represents a practical compromise between temporal resolution and computational tractability. This mesh is fine enough to support nontrivial pulse shaping, while keeping the horizon short enough for repeated multilevel rollouts to remain feasible.

A second important trade-off concerns policy stability versus exploration. In the present codebase, the stochastic policy is Gaussian in the raw action space, and therefore the parameter \texttt{init\_log\_std} directly controls the initial dispersion of sampled controls. If this value is too large, the early policy explores too broadly and frequently pushes actions into the saturation regime after the squashing map, which makes learning noisy and unstable. If it is too small, the policy explores only weakly and may fail to discover useful pulse trajectories within a reasonable number of iterations. A similar balance appears in the TRPO parameter \texttt{max\_kl}: large trust regions allow faster progress in principle, but also increase the risk of unstable policy jumps, whereas very small trust regions make learning conservative and slow. In the configuration adopted for the main experiments, these considerations lead to the nominal choices
\begin{equation}
\texttt{init\_log\_std}=-1.0,
\qquad
\texttt{max\_kl}=0.01,
\label{eq:default_exploration_trustregion}
\end{equation}
together with the standard TRPO settings
\begin{equation}
\gamma_{\mathrm{RL}}=0.99,
\qquad
\lambda=0.97,
\qquad
\texttt{cg\_iters}=10,
\qquad
\texttt{damping}=0.1.
\label{eq:default_trpo_auxiliary_parameters}
\end{equation}
These values are not presented as globally optimal, but as a stable regime that proved adequate for the representative target and remained close to standard TRPO practice.

The stopping threshold and batch size introduce a further methodological compromise. A loose termination criterion reduces training time, but risks accepting controls that are only superficially satisfactory. A stricter threshold is scientifically preferable, since it forces the policy to search for genuinely low-cost solutions, but it is also harder to satisfy consistently within a fixed training budget. In the present study, the default early-stopping threshold is set to
\begin{equation}
C_{\mathrm{term}}=0.15,
\label{eq:default_termination_cost}
\end{equation}
which is intentionally demanding and therefore avoids premature convergence toward mediocre solutions. On the data-collection side, the reference paper uses updates based on \(20000\) episodes per batch \cite{Niu2019}. In the present implementation, this value is retained for the main paper-matched runs, while lighter exploratory sweeps may use \(10000\) episodes per batch when computational cost must be reduced. In the same spirit, full comparison runs are allowed to continue for up to \(100\) policy iterations, whereas shorter exploratory searches may use much smaller iteration budgets.

The training-noise setting is fixed in direct continuity with the reference framework. During stochastic training, Gaussian control perturbations with standard deviation
\begin{equation}
\sigma_{\mathrm{train}} = 1\,\mathrm{MHz}
\label{eq:default_training_noise}
\end{equation}
are injected into the relevant control channels, consistently with the noisy-environment protocol described in the original paper \cite{Niu2019}. This choice is important not only because it preserves comparability with the reference results, but also because it defines the robust-training regime used later in the chapter. The broader robustness evaluation still spans a larger interval of noise strengths, but the value in Eq.~\eqref{eq:default_training_noise} is the default operating point at which the RL controller is exposed to stochastic perturbations during learning.

Taken together, these considerations define the default training setup used in the remainder of the empirical chapter:
\begin{equation}
\Delta t = 2\,\mathrm{ns},\quad
T_{\max}=180\,\mathrm{ns},\quad
\texttt{init\_log\_std}=-1.0,\quad
\texttt{max\_kl}=0.01,\quad
C_{\mathrm{term}}=0.15,
\label{eq:default_training_configuration_summary}
\end{equation}
with \(\gamma_{\mathrm{RL}}=0.99\), \(\lambda=0.97\), \(10\) conjugate-gradient iterations, damping \(0.1\), and a stochastic training noise of \(1\,\mathrm{MHz}\). Larger batches of \(20000\) episodes are used when staying as close as possible to the paper is important, while \(10000\)-episode batches are reserved for lighter preliminary sweeps. This configuration is not claimed to be uniquely optimal, but it provides a coherent and transparent operating regime under which later comparisons can be interpreted without ambiguity.

It is worth stressing once more that the present subsection concerns only \emph{training} hyperparameters. The architecture of the policy and value networks is deliberately held fixed here, so that the effect of optimization settings is not confounded with the representational capacity of the actor--critic model. Likewise, the UFO weights are not tuned in this subsection, because they are part of the physical scalarization of the control problem rather than ordinary optimizer settings. These two aspects are therefore deferred to the next subsections, where they can be analyzed on their own terms.

\subsection{Neural-network size sweep for policy and value functions}
\label{subsec:nn_size_sweep}

In the original UFO framework, both the policy and value functions are represented by fully connected neural networks with three hidden layers of sizes \(64\), \(32\), and \(32\) \cite{Niu2019}. Since the present implementation also relies on a continuous-action actor--critic TRPO pipeline, the architecture of these networks is not a negligible detail. The hidden-layer layout directly affects the representational capacity of the policy, the stability of the value-function approximation, the efficiency of exploration in the continuous control space, and ultimately the quality of the learned gate-control strategy. For this reason, the choice of network size should be treated separately from the tuning of training hyperparameters and from the selection of the UFO cost weights.

The purpose of the present subsection is therefore to assess whether modest changes in network width and depth materially affect performance in the current implementation. Importantly, this study is not intended as an exhaustive architecture search over a large design space. Rather, it is a restricted sweep aimed at answering a narrower question: within a family of architectures of comparable scale, does increasing width or depth improve the learned balance between control quality, leakage suppression, and robustness? This question is especially relevant because the reference paper motivates the use of neural networks as compact representations of non-local regularities in successful control trajectories \cite{Niu2019}. If that interpretation is correct, then architecture should matter, but not necessarily in a monotonic way: larger networks may provide more expressive power, yet also introduce harder optimization and greater seed sensitivity.

The sweep was performed on the same representative target gate used for the training-parameter study,
\begin{equation}
U_{\mathrm{rep}} = N(2.2,2.2,\pi/2),
\label{eq:nn_sweep_representative_target}
\end{equation}
and under a fixed reduced-budget training protocol designed for model selection. More precisely, all runs used noise-optimized TRPO with Gaussian training noise of standard deviation \(1\,\mathrm{MHz}\), timestep \(\Delta t=2\,\mathrm{ns}\), maximum gate duration \(T_{\max}=180\,\mathrm{ns}\), trust-region radius \(\texttt{max\_kl}=0.01\), initial exploration scale \(\texttt{init\_log\_std}=-1.0\), termination threshold \(C_{\mathrm{term}}=0.4\), and the nominal UFO weights
\begin{equation}
(\chi,\beta,\mu,\kappa)=(10,10,0.2,0.1).
\label{eq:nn_sweep_default_weights}
\end{equation}
Each architecture was trained for \(20\) TRPO iterations with \(5000\) episodes per batch and evaluated over \(3\) random seeds. The tested hidden-layer layouts were
\begin{equation}
(64,64,64),\qquad
(64,64,64,32),\qquad
(128,64,32),\qquad
(128,64,32,32).
\label{eq:tested_hidden_sizes}
\end{equation}
It should be noted explicitly that the exact paper architecture \((64,32,32)\) was not part of this restricted sweep. Consequently, the present study should be interpreted as a comparison among nearby thesis-side alternatives rather than as a direct empirical rejection of the reference architecture.

To compare the candidate networks, several criteria were considered simultaneously. The primary selection metric was the best evaluation cost reached during training, since this is the quantity most directly aligned with the UFO objective. However, this scalar alone is not sufficient for a meaningful choice. A network could achieve a favorable total cost by trading leakage against fidelity or by settling into an unnecessarily long runtime. For this reason, the architecture comparison also tracked best evaluation fidelity, best evaluation leakage, effective evaluation runtime, and robustness under noisy deployment through the average fidelity at \(\sigma = 1\,\mathrm{MHz}\). In this way, the architecture was not chosen solely by one scalar score, but by its overall position in the multi-objective trade-off relevant to the control problem.

The results of the restricted sweep show a clear but not entirely trivial pattern. Among the four tested candidates, the architecture
\begin{equation}
(64,64,64)
\label{eq:best_architecture_restricted_sweep}
\end{equation}
achieves the best overall mean evaluation cost,
\begin{equation}
\overline{C}_{\mathrm{eval}} = 0.3584,
\label{eq:best_architecture_cost}
\end{equation}
and also the highest mean best-evaluation fidelity,
\begin{equation}
\overline{F}_{\mathrm{eval}} = 0.9941.
\label{eq:best_architecture_fidelity}
\end{equation}
At the same time, it gives the highest mean robustness score at \(\sigma=1\,\mathrm{MHz}\),
\begin{equation}
\overline{F}_{\sigma=1\,\mathrm{MHz}} = 0.9637,
\label{eq:best_architecture_robustness}
\end{equation}
with a mean evaluation runtime of approximately
\begin{equation}
\overline{T}_{\mathrm{eval}} = 106.7\,\mathrm{ns}.
\label{eq:best_architecture_time}
\end{equation}
This is an important result because it indicates that, within the explored subset, moderate widening of the network improves the optimization outcome without requiring extra depth.

The deeper alternatives do not improve this picture. The architecture \((64,64,64,32)\) performs slightly worse in all main aggregated metrics, with a higher mean evaluation cost and a lower mean robust fidelity. The architecture \((128,64,32)\) remains close in mean cost, but displays a somewhat less favorable robustness profile and larger seed-to-seed variability. The deepest tested model, \((128,64,32,32)\), is particularly instructive: it achieves the \emph{lowest mean leakage},
\begin{equation}
\overline{L}_{\mathrm{eval}} = 3.56\times 10^{-4},
\label{eq:lowest_leakage_architecture}
\end{equation}
but at the price of the \emph{worst} mean evaluation cost, the lowest mean fidelity, the longest mean runtime, and the weakest mean robustness among the tested candidates. This indicates that minimizing leakage in isolation is not an adequate architecture-selection principle. A more expressive or deeper network may learn a comparatively conservative control strategy with slightly cleaner multilevel confinement, yet still fail to produce the best overall balance among the terms entering the UFO objective.

This interpretation is reinforced by the qualitative structure of the trade-off plots. The leakage boxplot is useful because it shows that the lowest-leakage architecture is not the one with the best total control quality. The robustness boxplot shows that the \(64\text{-}64\text{-}64\) architecture is not only strong on average, but also competitive in terms of seed-level consistency. Finally, the Pareto-style scatter plot of leakage versus robustness gives perhaps the clearest visual summary: the architectures do not lie on a simple monotonic frontier where lower leakage automatically implies higher robustness. Instead, the sweep suggests that the best architecture is the one occupying the most favorable compromise region of the leakage--robustness plane, rather than the one that optimizes either axis independently.

A further methodological point should be emphasized. Because the sweep was carried out under a reduced training budget, namely \(20\) policy iterations and \(5000\) episodes per batch, the resulting ranking should be interpreted as a comparison of \emph{practical trainability} under limited optimization time rather than as a proof of asymptotic superiority under full convergence. This distinction matters. A larger architecture may in principle catch up under a much longer training budget, but if it is consistently slower, noisier, or less reliable in the practical regime relevant to the rest of the thesis, then it is still a less attractive default choice. For the purposes of the present empirical chapter, the relevant question is therefore not which architecture might eventually win under unlimited compute, but which one provides the best and most stable control quality under the realistic budget used for the experiments.

On the basis of this restricted sweep, the architecture
\begin{equation}
(64,64,64)
\label{eq:selected_architecture_main_experiments}
\end{equation}
is adopted as the default policy/value hidden-layer configuration for the subsequent thesis-side experiments. This choice is motivated by its best mean evaluation cost, best mean fidelity, and strongest average robustness among the tested candidates, together with a competitive runtime and acceptable leakage. At the same time, this conclusion should be phrased carefully. Since the exact paper architecture \((64,32,32)\) was not included in the tested subset, the present result does not establish that the reference layout is suboptimal in absolute terms. It only shows that, within the explored family of nearby alternatives, \((64,64,64)\) is the most effective architecture for the current implementation.

For this reason, the present subsection supports two distinct conclusions. First, architecture choice is not neutral: changes in width and depth do measurably affect the final control trade-off. Second, the dependence on architecture is not monotonic: increasing network size does not automatically improve control quality, and in fact the deepest tested model performs worst in the aggregated objective despite having the lowest mean leakage. This is precisely why the architecture study is useful. It prevents the later experiments from relying on an arbitrary actor--critic layout, while also showing that the dominant effects observed in the rest of the chapter are unlikely to be artifacts of an especially pathological network choice.

\subsection{Sensitivity study of the UFO cost-function weights}
\label{subsec:ufo_weight_sweep}

The choice of the UFO coefficients deserves a separate study because, unlike ordinary optimizer hyperparameters, these quantities are part of the physical definition of the control problem itself. As mentioned in \ref{subsec:ufo_cost_function}, the total objective is written as
\begin{equation}
C(\chi,\beta,\mu,\kappa)
=
\chi\bigl[1-F(U(T))\bigr]
+
\beta L_{\mathrm{tot}}
+
\mu P_{\mathrm{bdry}}
+
\kappa T,
\label{eq:ufo_cost_recalled_weight_sweep}
\end{equation}
where \(\chi\) controls the weight of gate infidelity, \(\beta\) penalizes accumulated leakage, \(\mu\) enforces boundary regularity, and \(\kappa\) penalizes runtime \cite{Niu2019}. The reference paper reports that these coefficients were selected by grid search and then fixed uniformly across the target gates considered, with the final choice
\begin{equation}
\chi=\beta=10,
\qquad
\mu=0.2,
\qquad
\kappa=0.1.
\label{eq:paper_ufo_weights_recalled}
\end{equation}
For the purposes of the present thesis, this makes the weight study methodologically important in two complementary senses. On the one hand, it allows one to test whether the balance adopted in the paper is also reasonable in the present independent implementation. On the other hand, it clarifies which aspects of the observed performance are due to the optimizer and which are instead induced directly by the physical scalarization of the control objective. 

The sweep was performed on the same representative target gate used in the previous parameter-selection stages,
\begin{equation}
U_{\mathrm{rep}} = N(2.2,2.2,\pi/2),
\label{eq:ufo_weight_representative_target}
\end{equation}
under a fixed reduced-budget noise-optimized TRPO configuration. More precisely, the training setup was
\begin{equation}
\Delta t = 2\,\mathrm{ns},
\qquad
T_{\max}=180\,\mathrm{ns},
\qquad
\texttt{max\_kl}=0.005,
\qquad
\texttt{init\_log\_std}=-1.0,
\label{eq:ufo_weight_fixed_training_setup}
\end{equation}
with \(5\) TRPO iterations, \(10000\) episodes per batch, \(8\) workers, termination threshold \(0.15\), Gaussian training noise of standard deviation \(1\,\mathrm{MHz}\), and the paper-matched network architecture \(64\text{-}32\text{-}32\). Two random seeds were used for each configuration. The explored coefficient ranges were
\begin{equation}
\chi \in \{5,10,20\},
\qquad
\beta = 10,
\qquad
\mu \in \{0.1,0.2,0.4\},
\qquad
\kappa \in \{0.05,0.1,0.2,0.4\},
\label{eq:ufo_weight_sweep_ranges}
\end{equation}
for a total of \(3\times 1\times 3\times 4 = 36\) weight configurations and \(72\) runs once the two seeds are included.

The choice to keep \(\beta\) fixed deserves an explicit comment. Since the goal of the present experiment is to understand how the optimizer reacts to changing the balance between fidelity, boundary regularity, and runtime \emph{relative} to leakage suppression, it is natural to treat \(\beta\) as a reference scale and vary the other coefficients around it. In this sense, the sweep should be interpreted primarily through the ratios \(\chi/\beta\), \(\mu/\beta\), and \(\kappa/\beta\), rather than as a completely unconstrained four-dimensional search. This restriction also keeps the interpretation manageable: if all four coefficients were varied simultaneously, it would become much harder to isolate which part of the scalarization is actually responsible for a given change in behavior.

A second methodological point is even more important: the \emph{best evaluation cost} is not an informative global selection criterion across this sweep. Indeed, different values of \(\chi\), \(\mu\), and \(\kappa\) define different scalarizations of the same physical control problem, so the numerical value of \(C(\chi,\beta,\mu,\kappa)\) is not directly comparable across configurations. This is not just a formal issue. In the present experiment, all runs terminate at the same evaluation time of \(180\,\mathrm{ns}\), so increasing \(\kappa\) contributes mainly a nearly rigid additive shift to the total cost without necessarily improving the learned control in any physically meaningful sense. For this reason, the search-script primary metric is retained only as an internal bookkeeping quantity, whereas the physically relevant comparison is based instead on nominal fidelity, best evaluation leakage, and robustness average fidelity at \(\sigma=1\,\mathrm{MHz}\).

Under this interpretation, the first clear result concerns the fidelity weight \(\chi\). The robustness boxplot in Fig.~\ref{fig:ufo_weight_sweep_robustness} shows that \(\chi=5\) leads to a visibly broader and lower-performing distribution, with several runs collapsing into a poor-robustness regime around \(0.81\)--\(0.83\). By contrast, \(\chi=10\) and \(\chi=20\) both yield substantially tighter and higher robustness distributions, concentrated around \(0.92\)--\(0.95\). This indicates that the low-fidelity-penalty regime underweights the logical-accuracy term too strongly relative to the other penalties. In other words, if \(\chi\) is chosen too small compared with \(\beta\), the optimizer is allowed to settle into controls that may be acceptable in terms of leakage or boundary behavior, yet remain clearly inferior in actual gate quality. Increasing \(\chi\) to the paper scale already removes most of this failure mode, while a further increase to \(\chi=20\) produces only a marginal additional gain.

The behavior of the boundary weight \(\mu\) is more subtle and, in some sense, more interesting. The sweep reveals a clear non-monotonic pattern. The intermediate value
\begin{equation}
\mu = 0.2
\label{eq:mu_intermediate_best}
\end{equation}
produces the most favorable median trade-off, with high robustness and good nominal fidelity. Reducing the penalty to \(\mu=0.1\) remains viable, but leads on average to slightly larger leakage and a somewhat less favorable robustness profile. Increasing it to \(\mu=0.4\), however, makes the distribution strongly bimodal: for some values of \(\chi\) it yields some of the best runs of the entire sweep, while for others it produces a catastrophic degradation in robustness. This is clearly visible in the robustness boxplot, where \(\mu=0.4\) generates the widest spread and the strongest lower-tail failures. The physical interpretation is that boundary regularization is beneficial only up to a point. A moderate boundary penalty encourages smoother pulse endpoints and cleaner logical/Fock basis matching, but an overly strong one can overconstrain the optimizer and distort the balance with the fidelity term, thereby harming the overall control quality rather than improving it.

This interpretation is reinforced when leakage is examined separately. Figure~\ref{fig:ufo_weight_sweep_leakage} shows that larger \(\mu\) tends, on average, to reduce the leakage proxy. In fact, the lowest-leakage region of the sweep is often associated with \(\mu=0.4\). However, this does \emph{not} translate automatically into the best robust controller. The reason is precisely that leakage is only one component of the total physical objective. In several runs, the stronger boundary penalty helps suppress leakage but simultaneously worsens the broader fidelity--robustness trade-off. This is therefore another concrete example of the multi-objective nature of the UFO framework: minimizing one term of the cost more aggressively does not guarantee a better overall control solution.

The runtime coefficient \(\kappa\) has the weakest effect in the present restricted-budget sweep. This is visible both in the robustness sensitivity plot and in the leakage sensitivity plot, where the four tested values
\(\kappa\in\{0.05,0.1,0.2,0.4\}\) produce distributions that are almost indistinguishable at the level of median performance. This weak dependence is not surprising once the evaluation protocol is taken into account. Since every run reaches the same best evaluation time of \(180\,\mathrm{ns}\), the runtime term does not act here as a strong discriminating mechanism on the learned stopping time, but mainly as a reweighting of the scalar objective. In other words, under this short and fixed-horizon model-selection regime, \(\kappa\) does not yet have enough room to reshape the control strategy as dramatically as \(\chi\) and \(\mu\) do. A stronger effect of \(\kappa\) is more likely to appear in the later runtime-family experiments, where gate duration is itself a central observable rather than an almost fixed boundary condition of the sweep.

The clearest global summary of the experiment is given by the Pareto-style leakage--robustness scatter in Fig.~\ref{fig:ufo_weight_sweep_pareto}. Two clusters stand out. The first corresponds to a favorable region with robustness around \(0.94\) and leakage in the low-\(10^{-4}\) to mid-\(10^{-4}\) range. The second corresponds to a clearly worse region with robustness around \(0.81\)--\(0.83\), associated mainly with the underweighted-fidelity settings. The color coding by \(\chi\) makes the main trend immediately visible: low \(\chi\) is overrepresented in the poor-robustness cluster, whereas the paper-scale and larger-\(\chi\) settings dominate the Pareto-favorable part of the plane. At the same time, the frontier is not determined by \(\chi\) alone. The best region is occupied jointly by configurations near
\begin{equation}
(\chi,\mu) \approx (10,0.2)
\qquad \text{and} \qquad
(\chi,\mu) \approx (20,0.4),
\label{eq:ufo_weight_frontier_region}
\end{equation}
with only weak dependence on \(\kappa\) within the explored range.

This last observation is crucial for the final choice. The numerically best average robustness is attained by the family of configurations centered around \((\chi,\beta,\mu)=(20,10,0.4)\), but the gain over the paper choice
\((10,10,0.2)\) is extremely small. In the present sweep, the paper configuration remains essentially on the same Pareto-favorable frontier, with very similar robustness and leakage, while being directly aligned with the scalarization reported in the original study \cite{Niu2019}. Since the improvement offered by the larger-\(\chi\), larger-\(\mu\) corner is marginal and the present sweep is based on only two seeds and a very small training budget, there is no strong methodological reason to abandon the paper weights for the main experiments. On the contrary, keeping the reference choice
\begin{equation}
(\chi,\beta,\mu,\kappa)=(10,10,0.2,0.1)
\label{eq:selected_ufo_weights_final}
\end{equation}
is preferable for two reasons: it remains empirically near-optimal in the present implementation, and it preserves the cleanest possible fair-comparison protocol with the original UFO framework.

The main conclusion of this subsection is therefore twofold. First, the cost weights matter, and they matter in a physically interpretable way: too small a fidelity weight degrades robustness substantially, while an excessively strong boundary penalty can reduce leakage at the price of destabilizing the broader control trade-off. Second, the paper scalarization is not only historically motivated, but also empirically reasonable in the present independent implementation. For this reason, the remainder of the empirical chapter will retain the reference UFO coefficients in Eq.~\eqref{eq:selected_ufo_weights_final}, unless explicitly stated otherwise.

\subsection{Final parameter configuration for the remainder of the chapter}
\label{subsec:final_parameter_configuration}

In light of the parameter studies discussed above, a single reference configuration is now fixed for the remainder of the empirical chapter. The purpose of this final choice is not to claim global optimality over all possible numerical settings, but to define a stable and transparent operating regime under which the subsequent experiments can be compared consistently.

For the training hyperparameters, the default setup is
\begin{equation}
\Delta t = 2\,\mathrm{ns},
\qquad
T_{\max}=180\,\mathrm{ns},
\qquad
\texttt{init\_log\_std}=-1.0,
\qquad
\texttt{max\_kl}=0.01,
\qquad
C_{\mathrm{term}}=0.15,
\label{eq:final_training_setup}
\end{equation}
with \(\gamma_{\mathrm{RL}}=0.99\), \(\lambda=0.97\), damping \(0.1\), and \(10\) conjugate-gradient iterations. Unless stated otherwise, the noise-optimized runs are trained with Gaussian control noise of standard deviation
\begin{equation}
\sigma_{\mathrm{train}} = 1\,\mathrm{MHz},
\label{eq:final_training_noise}
\end{equation}
in direct continuity with the reference framework \cite{Niu2019}.

For the actor--critic representation, the architecture sweep suggests that network choice is not neutral, but also that improvements are not monotonic with increasing width or depth. Since the restricted sweep identified
\begin{equation}
(64,64,64)
\label{eq:final_hidden_sizes_choice}
\end{equation}
as the most favorable architecture among the tested alternatives, this layout is adopted as the default thesis-side configuration for the subsequent experiments.

Finally, for the UFO scalarization, the sensitivity study shows that the absolute value of the total cost is not directly comparable across different weight configurations, and that the most meaningful comparison must instead be based on fidelity, leakage, and robustness. Under this interpretation, the reference-paper choice
\begin{equation}
(\chi,\beta,\mu,\kappa)=(10,10,0.2,0.1)
\label{eq:final_ufo_weights_choice}
\end{equation}
remains very close to the best trade-off region observed in the present sweep. For this reason, and in order to preserve the cleanest possible comparison with the original UFO framework, these values are retained as the default cost-function weights in the remainder of the chapter \cite{Niu2019}.

Unless explicitly stated otherwise, all later experiments therefore use the combined reference configuration defined by
Eqs.~\eqref{eq:final_training_setup}--\eqref{eq:final_ufo_weights_choice}. This choice ensures that the empirical comparisons reported in the following sections are not affected by experiment-specific retuning, and that the results remain directly interpretable within the methodological logic established by the reference paper.

\section{Single-target control optimization}
\label{sec:single_target_optimization}
\subsection{Why this target is representative}
\label{subsec:why_this_target_is_representative}

As a representative nominal benchmark, this study first focuses on the target gate
\begin{equation}
U_{\mathrm{target}}
=
N(2.2,2.2,\pi/2),
\label{eq:representative_target_gate}
\end{equation}
which belongs to the two-qubit family \(N(\alpha,\alpha,\gamma)\) introduced in the previous section. This choice is motivated by three considerations. 
\begin{itemize}
    \item First, it is a genuinely nontrivial target: it is neither close to the identity nor reducible to a particularly simple limiting case, and therefore provides a meaningful test of the optimizer's ability to coordinate entangling dynamics, local control, and leakage suppression within the full multilevel model. 
    \item Second, it is directly aligned with the original benchmark logic of the reference paper, which uses the same gate family as the main testbed for robustness and runtime analyses \cite{Niu2019}. For this reason, selecting a target of the form \(N(\alpha,\alpha,\pi/2)\) allows the present study to remain as close as possible to the original physical and algorithmic setting while still isolating one concrete case for detailed discussion. 
    \item Third, fixing a single representative gate provides a clean comparison between optimization strategies before moving to broader parameter sweeps: by holding the target fixed, differences in final fidelity, leakage, runtime, and training behavior can be attributed more directly to the optimizer itself rather than to variations in target-gate difficulty across the family. 
\end{itemize}
In this sense, Eq.~\eqref{eq:representative_target_gate} serves as the natural entry point for the empirical analysis, since it is sufficiently rich to be physically informative while remaining simple enough to support a transparent method-by-method comparison.

\subsection{Adam baseline results}
\label{subsec:adam_baseline_results}

As a first non-RL reference, the representative target
\begin{equation}
U_{\mathrm{target}} = N(2.2,2.2,\pi/2)
\label{eq:adam_representative_target}
\end{equation}
was optimized by direct gradient-based pulse search using Adam, that is, through a baseline in the spirit of standard direct quantum optimal-control methods acting directly in control space rather than through policy learning \cite{Khaneja2005,Koch2022}. 
The optimization was performed on the same multilevel gmon model and with the same UFO-inspired objective introduced in the previous sections, using a time discretization of \(\Delta t = 2\,\mathrm{ns}\), \(400\) Adam iterations for each candidate horizon, and a sweep over gate durations in the interval \(T\in[40,180]\,\mathrm{ns}\) with \(10\,\mathrm{ns}\) spacing. 
In order to keep the baseline aligned with the robustness-oriented setting of the paper, the optimization was carried out with the same nominal UFO weights \((\chi,\beta,\mu,\kappa)=(10,10,0.2,0.1)\) and with train-time Gaussian control noise of standard deviation \(1\,\mathrm{MHz}\), while the final metrics discussed in this subsection are recomputed in the nominal simulator.\footnote{This distinction is important in the present implementation. The internal optimizer summary selects \(T=60\,\mathrm{ns}\) as the minimum of the noisy optimization objective, whereas the deterministic post-training reevaluation identifies \(T=70\,\mathrm{ns}\) as the best nominal solution. Since the purpose of the present subsection is to establish the \emph{nominal} single-target benchmark, the \(70\,\mathrm{ns}\) solution is taken as the representative Adam baseline in the discussion below.}

Under this nominal reevaluation, the best Adam solution is obtained at
\begin{equation}
T_{\mathrm{Adam}}^{\star}=70\,\mathrm{ns},
\label{eq:adam_best_nominal_horizon}
\end{equation}
for which the final cost is
\begin{equation}
C_{\mathrm{Adam}}^{\mathrm{nom}} = 0.1576,
\label{eq:adam_best_nominal_cost}
\end{equation}
with nominal fidelity
\begin{equation}
F_{\mathrm{Adam}}^{\mathrm{nom}} = 0.99662,
\label{eq:adam_best_nominal_fidelity}
\end{equation}
total leakage
\begin{equation}
L_{\mathrm{Adam}}^{\mathrm{tot}} = 2.82\times 10^{-4},
\label{eq:adam_best_nominal_leakage}
\end{equation}
and boundary contribution
\begin{equation}
P_{\mathrm{bdry}}^{\mathrm{Adam}} = 4.28\times 10^{-3}.
\label{eq:adam_best_nominal_boundary}
\end{equation}
These values already indicate that direct pulse optimization is capable of finding a high-quality control for the representative target, with gate fidelity comfortably above \(99\%\) and leakage kept in the \(10^{-4}\) regime, which is precisely the scale expected from the leakage-aware control philosophy of the original UFO framework \cite{Niu2019}. At the same time, the result is scientifically useful not only because it is "good" but because the horizon sweep reveals \emph{why} it is good and where the main tradeoffs arise.

The dependence on the gate duration is particularly instructive. Very short horizons, such as \(40\,\mathrm{ns}\) and \(50\,\mathrm{ns}\), perform poorly: the nominal fidelities drop to \(0.7255\) and \(0.8818\), while the corresponding boundary penalties remain very large, respectively \(0.399\) and \(0.357\). By contrast, the leakage itself stays in the same \(10^{-4}\) range throughout this short-time regime. This means that the main failure mode of the ultra-short solutions is \emph{not} an isolated explosion of leakage alone, but the inability to satisfy simultaneously the accuracy and boundary requirements of the UFO objective under a strongly time-compressed pulse. This behavior is fully consistent with the speed--leakage and speed--smoothness tension discussed in Section~\ref{subsec:speed_leakage_tradeoff}: in weakly anharmonic superconducting systems, shortening the pulse too aggressively broadens the effective control spectrum and makes it more difficult to maintain both clean logical evolution and well-behaved pulse boundaries \cite{Motzoi2009,Koch2022}. The sharp transition from \(60\,\mathrm{ns}\) to \(70\,\mathrm{ns}\) is especially revealing in this respect. Over this interval, the fidelity increases from \(0.9821\) to \(0.9966\), the boundary term collapses from \(9.75\times10^{-2}\) to \(4.28\times10^{-3}\), and the total nominal cost drops from \(0.3806\) to \(0.1576\), whereas the leakage decreases only moderately. This indicates that, near the optimum, the decisive improvement is not merely "less", but the fact that the pulse becomes sufficiently long to realize the target unitary while also respecting the boundary structure required by the UFO cost.

The long-horizon side of the sweep is equally informative. Beyond approximately \(70\)--\(90\,\mathrm{ns}\), the fidelity remains uniformly high and varies only weakly; for example, at \(150\,\mathrm{ns}\) the Adam baseline reaches
\begin{equation}
F_{\mathrm{Adam}}^{\mathrm{nom}}(150\,\mathrm{ns}) = 0.99708,
\end{equation}
slightly above the \(70\,\mathrm{ns}\) value, while the leakage decreases further to \(6.82\times10^{-5}\). Nevertheless, the total cost rises again to \(0.2851\). This is an important point for the logic of the thesis. If one were to judge the control only by fidelity, the longer pulse would appear at least competitive, and perhaps even slightly preferable. However, once the full UFO objective is taken seriously, the improvement in fidelity and leakage is too small to compensate for the increased runtime. In other words, the Adam sweep gives a direct numerical illustration of the multi-objective nature of the framework introduced in Section~\ref{subsec:ufo_cost_function}: the best solution is not the one with the absolute highest fidelity, nor the one with the absolute smallest leakage, but the one that best balances accuracy, leakage suppression, boundary regularity, and total gate time. This is precisely the type of behavior one would expect in realistic superconducting-qubit control, where gate design is not a single-objective search but a compromise between physically competing desiderata \cite{Niu2019,Koch2022}.

A further useful benchmark emerges when this nominal Adam runtime is compared to the circuit-based reference used in the original paper. For the family \(N(\alpha,\alpha,\gamma)\), Niu \emph{et al.} estimate an optimal synthesized implementation time of approximately \(215\,\mathrm{ns}\), obtained from a depth-seven decomposition with three entangling gates and five single-qubit gates \cite{Niu2019}. Relative to this reference, the representative Adam solution at \(70\,\mathrm{ns}\) is already faster by a factor of about
\begin{equation}
\frac{215\,\mathrm{ns}}{70\,\mathrm{ns}} \approx 3.1,
\end{equation}
even before introducing policy optimization or transfer learning. This does \emph{not} yet prove the stronger runtime claims of the RL framework across the whole gate family, but it is a useful early indication that direct analog optimization on the present hardware model can already exploit structure unavailable to a fixed compiled-gate implementation.

Although the systematic robustness study is deferred to Section~\ref{subsec:robustness_under_noise}, the Adam baseline also provides a first quantitative reference point in the noisy setting. For the control plan selected by the noisy training objective at \(60\,\mathrm{ns}\), the robustness evaluation at \(\sigma_{\mathrm{noise}}=1\,\mathrm{MHz}\) yields an average fidelity of \(0.9686\), an average gate fidelity of \(0.9607\), and a fidelity variance of \(1.75\times10^{-4}\). These values are already respectable for a direct gradient baseline, but they also make clear why a stronger robustness-oriented method is desirable: they set the scale that the nominal and noise-optimized TRPO controllers will later be asked to surpass. In this sense, the Adam baseline performs exactly the role it should in the empirical chapter: it is not a strawman, but a meaningful non-RL benchmark that captures the main structure of the control problem while leaving clear room for improvement in robustness and consistency under stochastic perturbations.

A preliminary family-wide Adam sweep with fixed \(\gamma=\pi/2\) further reinforces this interpretation. Even without transfer learning, the performance of the direct optimizer is markedly non-uniform across the gate family \(N(\alpha,\alpha,\pi/2)\). The best results cluster around the central region \(\alpha\approx 1.4\)--\(1.7\), where the optimizer consistently selects \(40\,\mathrm{ns}\) as the best horizon, reaches nominal fidelities above \(0.9987\), and retains noisy average fidelities around \(0.987\)--\(0.989\) with very small fidelity variance of order \(10^{-5}\). By contrast, near the edges of the interval, i.e. for \(\alpha\approx 0\) or \(\alpha\approx \pi\), the best horizon shifts to \(70\,\mathrm{ns}\), the nominal fidelity drops to about \(0.94\), and the noisy average fidelity falls to about \(0.936\)--\(0.938\), with variance increasing by roughly one order of magnitude. This shows that the difficulty of the gate family is highly structured already at the level of direct gradient optimization. Not all members of \(N(\alpha,\alpha,\pi/2)\) are equally easy to realize within the same bandwidth-limited multilevel control ansatz, and this fact will later be important when interpreting the runtime curves obtained with RL and transfer learning. Put differently, the Adam sweep provides a useful "zero" baseline for distinguishing between intrinsic gate-family geometry and genuine algorithmic gains due to policy optimization.

\subsection{Nominal TRPO results}
\label{subsec:nominal_trpo_results}

The second benchmark on the representative target
\begin{equation}
U_{\mathrm{target}} = N(2.2,2.2,\pi/2)
\label{eq:nominal_trpo_target}
\end{equation}
is obtained with the TRPO-based reinforcement-learning controller introduced in Chapter~3, but trained in the \emph{nominal} environment, i.e. without stochastic perturbations injected during training. In the present implementation, this corresponds to setting \texttt{noise\_optimized = false}, so that the policy is optimized against the deterministic multilevel gmon dynamics discussed in Section~\ref{subsec:paper_rl_formulation}, even though the codebase retains the same general training interface used later for the noisy runs. The controller is trained with the same physical model and UFO weights used in the Adam baseline, namely \((\chi,\beta,\mu,\kappa)=(10,10,0.2,0.1)\), a time discretization \(\Delta t = 2\,\mathrm{ns}\), a maximum admissible gate time \(T_{\max}=180\,\mathrm{ns}\), and a termination threshold \(C_{\mathrm{term}}=0.3\). On the algorithmic side, the nominal TRPO run uses batches of \(5000\) episodes, \(115\) policy updates, trust-region radius \(\texttt{max\_kl}=0.01\), discount factor \(\gamma_{\mathrm{RL}}=0.99\), GAE parameter \(\lambda=0.97\), and fully connected policy/value networks with hidden sizes \(64\text{-}32\text{-}32\), consistent with the architecture described by Niu \emph{et al.} \cite{Niu2019}.

Under deterministic evaluation, the best policy found in this nominal setting achieves
\begin{equation}

\qquad
F_{\mathrm{TRPO}}^{\mathrm{nom}} = 0.99415,
\qquad
L_{\mathrm{TRPO}}^{\mathrm{nom}} = 1.60\times10^{-3},

\end{equation}
with an effective gate duration
\begin{equation}
T_{\mathrm{TRPO}}^{\mathrm{nom}} = 116\,\mathrm{ns}.

\end{equation}
The same deterministic rollout also reports
\begin{equation}

\qquad \text{at} \qquad

\label{eq:nominal_trpo_min_cost_time}
\end{equation}
which indicates that, unlike the Adam baseline where the horizon is fixed externally and selected by sweep, the nominal TRPO controller learns the control trajectory inside a larger admissible window and reaches its best stopping point well before the hard upper bound of \(180\,\mathrm{ns}\). This is an important structural difference between the two methods: Adam solves a family of fixed-horizon pulse-optimization problems, whereas TRPO explores a variable-length sequential control strategy under a cost-based stopping criterion. In that sense, the nominal TRPO result should not be interpreted simply as "another" point, but as the output of a policy that co-optimizes pulse shape and effective stopping time within the same RL formulation \cite{Niu2019}.

From the nominal-performance viewpoint, this result is already strong in absolute terms. A deterministic fidelity above \(99.4\%\) on the full multilevel model confirms that the policy-gradient formulation is able to learn a nontrivial high-quality analog control for the representative target. Moreover, the learned runtime of \(116\,\mathrm{ns}\) remains substantially below the \(215\,\mathrm{ns}\) synthesized-gate reference quoted in the original paper for the optimal circuit decomposition of the same target family \cite{Niu2019}. In other words, even before introducing noise-aware training, the nominal TRPO controller already supports the more general thesis-level claim that direct analog control can exploit hardware-native structure to outperform standard compiled implementations in runtime. The gain is, however, more moderate than in the strongest analog-control cases emphasized by the paper: for the present target, the reduction is approximately
\begin{equation}
\frac{215\,\mathrm{ns}}{116\,\mathrm{ns}} \approx 1.85,
\label{eq:nominal_trpo_vs_synthesis_factor}
\end{equation}
which is significant, but far from the order-of-magnitude speedups later reported for the noise-optimized runtime sweeps across the broader gate family \cite{Niu2019}.

At the same time, when this nominal TRPO solution is compared with the Adam baseline of the previous subsection, an instructive picture emerges. The best nominal Adam solution at \(70\,\mathrm{ns}\) achieved a lower total cost, slightly higher fidelity, significantly smaller leakage, and a shorter runtime. More explicitly, TRPO is worse than Adam on this single deterministic target both in total cost (\(0.2982\) vs. \(0.1576\)) and in gate duration (\(116\,\mathrm{ns}\) vs. \(70\,\mathrm{ns}\)), while its leakage is larger by roughly a factor of
\begin{equation}
\frac{1.60\times10^{-3}}{2.82\times10^{-4}} \approx 5.7.
\label{eq:nominal_trpo_leakage_ratio}
\end{equation}
The fidelity difference is much smaller in absolute terms, but still present:
\begin{equation}
F_{\mathrm{Adam}}^{\mathrm{nom}} - F_{\mathrm{TRPO}}^{\mathrm{nom}}
\approx 2.47\times10^{-3}.
\label{eq:nominal_fidelity_gap_adam_trpo}
\end{equation}
This comparison is scientifically important, because it prevents an over-interpretation of the RL method. On the representative clean target, nominal TRPO does \emph{not} dominate a strong direct pulse optimizer. Instead, the deterministic single-target benchmark suggests that, when the model is known accurately and the objective is smooth and fully differentiable, direct gradient-based optimization remains highly competitive, and in the present case superior, in the purely nominal regime. This is broadly consistent with the standard optimal-control literature, where direct gradient methods such as GRAPE are known to be particularly effective on fixed-target closed-system pulse-design problems \cite{Khaneja2005,Koch2022}. 

This observation is not a weakness of the RL framework; rather, it clarifies what the RL framework should be expected to contribute. The original paper does not primarily claim that TRPO is universally better than direct optimization on every nominal benchmark. Its stronger claim is instead that a policy-based method trained in the appropriate stochastic environment can improve \emph{robustness} and, in some hardware-aligned families, \emph{runtime efficiency} relative to simpler baselines \cite{Niu2019}. The present nominal-clean result is therefore valuable precisely because it isolates what TRPO does \emph{without} the noise-aware component: it learns a high-fidelity analog gate and a nontrivial variable-time control strategy, but it does not yet exhibit an intrinsic nominal advantage over a well-tuned direct optimizer on this representative deterministic target. This makes the next subsection conceptually much stronger, because any improvement observed there can be attributed more specifically to stochastic training rather than to the mere use of reinforcement learning.

A second useful insight concerns the qualitative role of the UFO scalarization. The nominal TRPO controller reaches a reasonable deterministic fidelity, but it does so at a larger runtime and with substantially more leakage than the best Adam solution. This suggests that, in the clean setting, the policy has found a more conservative but less sharply optimized compromise in the multi-objective landscape. Put differently, the RL controller appears able to discover acceptable controls within the admissible region of the UFO cost, but in the absence of explicit noise pressure it has less incentive to carve out the particularly efficient low-leakage solution that a direct horizon-conditioned optimizer can reach on the same target. This interpretation is fully consistent with the control-theoretic discussion in Sections~\ref{subsec:classical_qoc_methods}, \ref{subsec:why_rl_is_relevant}, and \ref{subsec:reward_design_bridge_physics_rl}: policy optimization is especially attractive when sequential structure, uncertainty, and robustness matter, but it need not be the most aggressive optimizer of a single deterministic pulse-shaping problem.

Finally, the nominal TRPO experiment also motivates a careful reading of training curves. Because the controller is evaluated after every policy update and the code stores both the current evaluation cost and the time \(t_{\min}\) at which the minimum rollout cost is reached, the most informative training visualization is not simply fidelity versus iteration. A better representation is one that jointly tracks the evolution of evaluation cost, fidelity, leakage, and learned stopping time. In the present case, such a plot would show that the policy quickly enters a high-fidelity regime but continues to trade off leakage and runtime within the admissible \(180\,\mathrm{ns}\) window, eventually settling on a best deterministic solution around the \(116\,\mathrm{ns}\) scale. This makes the nominal TRPO subsection a natural bridge between the fixed-horizon analysis of the Adam baseline and the explicitly robustness-oriented analysis of the next subsection.

\subsection{Comparative discussion}
\label{subsec:comparative_discussion_single_target}

Taken together, the Adam and nominal-TRPO results clarify an important point about the representative single-target problem: the reinforcement-learning formulation is already \emph{competitive} in the nominal setting, but it is not yet \emph{superior} to a strong direct pulse optimizer on this deterministic benchmark. This distinction is essential for the interpretation of the remainder of the chapter. The nominal TRPO controller reaches a high-quality solution for
\begin{equation}
U_{\mathrm{target}} = N(2.2,2.2,\pi/2),
\label{eq:comparative_target_gate}
\end{equation}
with fidelity above \(99.4\%\) and runtime well below the synthesized-gate reference of about \(215\,\mathrm{ns}\) discussed in the original paper \cite{Niu2019}. However, when the same target is solved by direct gradient-based optimization with a horizon sweep, the Adam baseline achieves a clearly better nominal compromise among the quantities entering the UFO objective.

Using the best deterministic solutions identified in the previous two subsections, the comparison can be summarized as follows.

This outcome is physically and methodologically plausible. As discussed in Sections~\ref{subsec:classical_qoc_methods} and \ref{subsec:reward_design_bridge_physics_rl}, direct gradient-based quantum optimal-control methods are particularly strong when the problem is fully model-based, deterministic, and differentiable, and when the objective concerns a fixed target gate rather than a distribution of related tasks \cite{Khaneja2005,Koch2022}. The present single-target benchmark lies precisely in that regime. Adam is allowed to optimize directly over the pulse parameters for each fixed horizon, and the horizon sweep then selects the best compromise afterward. TRPO, by contrast, optimizes a stochastic policy over sequential actions under a termination mechanism and must learn both the control law and the effective stopping structure within a single actor--critic pipeline. That extra flexibility is potentially valuable, but it also makes the nominal optimization problem less direct. For this reason, it is not surprising that, on one clean target, the direct optimizer can still produce the sharper deterministic solution.

At the same time, the comparison should not be misread as evidence against the RL formulation. First, nominal TRPO still performs well in absolute terms: a fidelity of \(0.99415\) at \(116\,\mathrm{ns}\) remains a strong analog-control result, and it already supports the broader thesis claim that direct pulse-level optimization can beat the standard synthesized-gate runtime on realistic multilevel hardware \cite{Niu2019}. Second, and more importantly, the nominal single-target benchmark is not where the main conceptual advantage of the RL framework is expected to appear. The discussion of robustness in Sections~\ref{subsec:why_robustness_is_difficult}, \ref{subsec:why_rl_is_relevant}, and \ref{subsec:stochastic_training_environment} already suggested that policy-based learning becomes especially attractive when the control problem is sequential, stochastic, and evaluated over ensembles of noisy trajectories rather than over one deterministic rollout. The present result is fully consistent with that logic: nominal TRPO does not dominate Adam on the clean benchmark, which means that any later advantage of the RL approach under noise cannot be dismissed as a trivial consequence of "RL" better at everything. Instead, it will have to be interpreted as a specific gain produced by the stochastic training environment and by the ability of the policy to internalize robustness-oriented regularities across trajectories, exactly as claimed in the original UFO paper \cite{Niu2019}.

A second important lesson concerns the role of the UFO scalarization itself. The single-target comparison makes clear that excellent nominal fidelity alone is not sufficient for a control to be considered preferable. The Adam and nominal-TRPO solutions both lie in a high-fidelity regime, yet they differ substantially in leakage and runtime. This confirms, in a concrete numerical setting, the conceptual point developed earlier in Sections~\ref{subsec:fidelity_runtime_metrics} and \ref{subsec:ufo_cost_function}: the relevant notion of success is not the maximization of one metric in isolation, but the identification of a balanced solution in a multi-objective landscape. In the representative target studied here, the direct optimizer reaches a better balance than nominal TRPO. The RL controller seems to settle into a broader, more conservative basin of acceptable solutions, whereas Adam succeeds in driving the pulse closer to the sharper low-runtime, low-leakage region of the landscape. This difference in optimization behavior will become important later, because noisy training may change which basin is actually preferable once robustness is included explicitly in the evaluation.

From the viewpoint of the overall chapter structure, this comparison also provides a clean logical transition to the next subsection. The question raised by the nominal benchmark is now sharply formulated: if direct gradient optimization already performs better on the deterministic single-target problem, why should one prefer the RL framework at all? The answer, if the main thesis of the reference paper is correct, must come from the behavior under stochastic perturbations. In other words, the present subsection establishes the necessary baseline against which the robustness claims of the framework can be judged. Adam is not a weak reference, but a strong nominal competitor. Therefore, if the noise-optimized RL controller later surpasses it in average fidelity and fidelity variance under control noise, that improvement will constitute meaningful evidence that robustness is not merely a side effect of analog optimization, but a property that is genuinely learned from the stochastic training environment \cite{Niu2019}.

\section{Robustness under stochastic control noise}
\subsection{Noise model}
\label{subsec:noise_model}
To assess robustness in a physically meaningful and methodologically consistent way, all controllers considered in this section are evaluated under the same stochastic control-error model, following the general logic of the original UFO framework \cite{Niu2019}. At the discrete time-step level, the nominal control trajectory is not implemented exactly, but is perturbed by additive Gaussian amplitude fluctuations. In the notation of the control vector introduced in Eq.~\eqref{eq:validation_control_vector}, this means that the realized control at time step \(t_k\) takes the schematic form
\begin{equation}
\bm{u}^{(\mathrm{noisy})}(t_k)
=
\bm{u}(t_k)+\bm{\xi}(t_k),
\label{eq:robustness_noisy_control_vector}
\end{equation}
where the noise term \(\bm{\xi}(t_k)\) is sampled independently at each time step from a zero-mean Gaussian distribution. In line with both the original paper and the present implementation, these fluctuations are applied not only to the directly actuated control amplitudes, namely the coupling pulse \(g(t)\), the detuning amplitudes \(\delta_j(t)\), and the microwave amplitudes \(f_j(t)\), but also to the anharmonicity-related parameter \(\eta\), which enters the Hamiltonian as part of the multilevel device model \cite{Niu2019}. This point is important: the noise model is not restricted to imperfect pulse delivery alone, but also probes sensitivity to uncertainty in a physically relevant Hamiltonian parameter.

At the Hamiltonian level, each noisy rollout is therefore generated by replacing the nominal control-dependent evolution with a perturbed one-step propagator,
\begin{equation}
U_{k+1}
=
\exp\!\left[
-i\Delta t\,
\bigl(
\hat{H}(t_k)+\delta \hat{H}(t_k)
\bigr)
\right]
U_k,
\label{eq:robustness_noisy_propagator}
\end{equation}


where \(\delta \hat{H}(t_k)\) collects the Gaussian perturbations induced by the noisy realization of the control parameters and of \(\eta\). The overall noise strength is parametrized by a standard deviation \(\sigma_{\mathrm{noise}}\), which is varied systematically in the robustness study below. In this way, robustness is not inferred from a single perturbed trajectory, but from a family of stochastic control channels of increasing severity.

A further methodological distinction must be made between \emph{training noise} and \emph{evaluation noise}. For the Adam baseline and the nominal TRPO controller, optimization is performed in the deterministic environment, and robustness is assessed only \emph{a posteriori} through noisy Monte Carlo rollouts. By contrast, for the noise-optimized TRPO agent, the same Gaussian perturbation model is already activated during training, so that the policy is learned in a stochastic environment rather than in a nominal one. This distinction is central for the interpretation of the results: if the noise-optimized TRPO controller later exhibits both higher average fidelity and lower fidelity variance under noisy evaluation, then the improvement cannot be attributed to reinforcement learning in general, but specifically to the fact that robustness has been learned from exposure to the stochastic environment itself \cite{Niu2019}.

The noisy performance of a given controller is then characterized statistically by repeated sampling. For each value of \(\sigma_{\mathrm{noise}}\), a fixed optimized control plan is executed over many independent noisy realizations, and the resulting empirical distribution of final fidelities is used to estimate both the average fidelity and the fidelity variance. In other words, the robustness metrics reported in the next paragraphs are not single-shot quantities, but Monte Carlo estimates over an ensemble of noisy trajectories,
\begin{equation}
\overline{F}(\sigma_{\mathrm{noise}})
=
\mathbb{E}_{\delta\bm{u}\sim\mathcal{N}(0,\sigma_{\mathrm{noise}})}
\left[
F\bigl(U(\bm{u}+\delta\bm{u})\bigr)
\right],
\label{eq:robustness_average_fidelity_section}
\end{equation}

\begin{equation}
\overline{F}(\sigma_{\mathrm{noise}})
=
\mathbb{E}_{\delta\bm{u},\,\delta\eta}
\left[
F\bigl(U(\bm{u}+\delta\bm{u};\,\eta+\delta\eta)\bigr)
\right],
\label{eq:robustness_average_fidelity_section}
\end{equation}

and
\begin{equation}
\mathrm{Var}(F)(\sigma_{\mathrm{noise}})
=
\mathbb{E}_{\delta\bm{u}}
\left[
\bigl(F-\overline{F}\bigr)^2
\right].
\label{eq:robustness_fidelity_variance_section}
\end{equation}
This evaluation protocol is particularly well suited to the present thesis, because it directly probes the central claim of the UFO framework: a useful control strategy should not only achieve high nominal fidelity, but should remain accurate and stable under realistic stochastic perturbations of the multilevel superconducting-qubit dynamics.

relative to the nominal TRPO solution across most of the explored range, especially once the noise
strength leaves the perturbative regime and the curves become more clearly separated. This is an
important result, because the two methods share the same policy-gradient machinery and differ mainly
in the environment seen during training. The lower variance of the noise-optimized controller therefore
cannot be explained by the mere use of reinforcement learning; rather, it indicates that exposure to
stochastic perturbations during training has produced a policy whose final performance is less sensitive
to random actuation errors from one realization to the next. In statistical terms, the learned policy
does not only preserve a higher mean performance, but also suppresses part of the dispersion induced
by the noisy channel.

was reported to achieve an approximately one-order-of-magnitude reduction in fidelity variance relative
to both the noise-free RL controller and the gradient baseline throughout the tested noise range
\cite{Niu2019}. The present implementation reproduces the same qualitative mechanism in a slightly more
nuanced form: the stochastic TRPO controller is again clearly more stable than the nominal TRPO
controller, confirming that robustness is indeed learned rather than accidentally inherited from the
nominal optimizer, but the Adam baseline remains exceptionally compact in variance. This suggests that,
in the present reproduction, the optimization landscape admits a short and relatively stiff Adam
solution that is statistically concentrated under noise, even though it sacrifices both nominal quality
and mean noisy fidelity. For the thesis, this is actually a useful result rather than a problem,
because it sharpens the interpretation: \emph{robustness} is multi-dimensional, and the most convincing
controller is not the one minimizing variance alone, but the one achieving the best trade-off between
high average fidelity and controlled dispersion.

For this reason, Fig.~\ref{fig:fidelity_variance_vs_noise} should be discussed jointly with
Fig.~\ref{fig:avg_fidelity_vs_noise}. Taken together, the two figures show that the noise-optimized
TRPO controller occupies the best overall compromise among the tested methods: it consistently attains
the highest average fidelity under stochastic perturbations, while also keeping the fidelity variance
well below that of the nominal TRPO controller. The Adam baseline remains an instructive counterexample:
its variance curve is favorable, but its mean-performance curve is systematically inferior. Hence, the
statistical evidence does not support the simplistic claim that the most robust controller is merely the
least fluctuating one; rather, it supports the stronger statement that robustness in the UFO sense is a
joint property of \emph{accuracy} and \emph{stability}, and that noisy-environment training is what most
effectively improves that joint property in the present study.


\subsection{Interpretation}
\label{subsec:robustness_interpretation}

Taken together, the average-fidelity and fidelity-variance curves provide the clearest empirical test of the
central claim of the present reproduction. The key question of this subsection is not whether noisy control
perturbations degrade all solutions (which they inevitably do), but whether training in a stochastic environment
induces a qualitatively different and more robust control strategy than training in a nominal one. From this
perspective, the most informative comparison is not the one between reinforcement learning and direct pulse
optimization in the abstract, but the one between the \emph{two} TRPO controllers: the nominal controller and
the noise-optimized controller. These two methods share the same architecture, the same policy-gradient
machinery, the same Hamiltonian model, and the same UFO objective; the decisive difference is that only one of
them is exposed during training to the Gaussian perturbation model described in
Sec.~\ref{subsec:noise_model}. The fact that the noise-optimized controller systematically attains higher average
fidelity and lower fidelity variance than the nominal TRPO solution therefore carries a very direct
interpretation: the observed robustness is not merely an incidental byproduct of the RL formalism, but a
property that is learned from the stochastic training environment itself \cite{Niu2019}.

This point is conceptually important for the thesis. In the nominal single-target benchmark of
Sec.~\ref{subsec:single_target_control_optimization}, the deterministic TRPO controller was shown to be
competitive but not superior to a strong direct Adam baseline. In that regime, the use of reinforcement learning
alone did not guarantee a better pulse. The robustness study changes the picture. Once performance is measured
under a family of noisy realizations rather than on a single deterministic rollout, the controller trained in the
stochastic environment becomes the one that preserves the largest average fidelity across the full explored noise
range, while also reducing the statistical dispersion relative to the nominal TRPO solution. This is exactly the
type of behaviour one would expect if the policy were not simply fitting one nominal pulse trajectory, but
instead learning a control strategy that is less sensitive to random perturbations of the underlying
multilevel Hamiltonian.

At the same time, the present data suggest a more nuanced conclusion than the strongest formulation reported in
the original UFO paper. In the reference work, the noise-optimized RL controller was reported to outperform both
the noise-free RL controller and the gradient-based baseline also in fidelity variance by approximately one order
of magnitude throughout the tested range \cite{Niu2019}. In the present reproduction, the comparison with the
Adam baseline is more subtle. The Adam solution remains the narrowest distribution in the strict variance sense,
that is, the least fluctuating controller under repeated noisy realizations. However, this lower variance is not
paired with the best mean performance: the same Adam baseline lies systematically below the noise-optimized TRPO
controller in average fidelity. For this reason, the present results do not support the simplistic conclusion that
robustness should be identified with variance minimization alone. Rather, they support a more physically relevant
statement: a robust controller must maintain \emph{both} high mean fidelity and controlled shot-to-shot
fluctuations, and under this joint criterion the noise-optimized TRPO controller provides the best overall
operating point among the three methods tested here.

This distinction is also meaningful from the viewpoint of control theory. The Adam baseline appears to produce a
"stiffer" solution: its noisy realizations remain more tightly clustered, but around a lower-fidelity mean.
The noise-optimized TRPO controller, by contrast, occupies a different region of the performance landscape. Its
output distribution is broader than Adam's, yet it is centered at a consistently better average fidelity across
the whole family of Gaussian perturbations considered in the robustness sweep. In practical terms, this means
that the stochastic TRPO agent is not merely learning to reduce dispersion; it is learning to shift the noisy
performance distribution toward a better robust mean while simultaneously avoiding the large variance growth
displayed by the nominal TRPO solution. This is arguably the more relevant notion of robustness for analog
quantum control, where one is not interested in a controller that is merely reproducible, but in one that remains
reproducibly \emph{accurate} under realistic perturbations.

A second important implication concerns the role of the stochastic environment itself. Because the evaluation
noise is not restricted to a single point but is swept continuously over
\(\sigma_{\mathrm{noise}}\in[0.1,3.5]\,\mathrm{MHz}\), the superiority of the noise-optimized TRPO controller cannot
be interpreted as a trivial overfitting to one fixed training noise realization. Instead, the controller trained
at the nominal paper setting of \(1\,\mathrm{MHz}\) displays a more graceful degradation across a broader family of
Gaussian noise strengths, which is precisely the kind of in-family generalization claimed in the original paper
\cite{Niu2019}. In this sense, the present results support the methodological thesis that policy-gradient
optimization in a stochastic environment can internalize useful regularities of noisy control trajectories and
convert them into improved robust performance at deployment time.

For the overall structure of the chapter, this subsection therefore serves as the empirical validation point of
the framework. The deterministic results of Sec.~\ref{subsec:single_target_control_optimization} established that
all three methods can produce meaningful high-quality controls on the representative gate
\(N(2.2,2.2,\pi/2)\), but did not by themselves justify the use of noisy-environment RL. The robustness curves do.
They show that once the control problem is evaluated in the physically relevant regime of stochastic actuation
errors, the decisive advantage of the RL framework is not simply that it optimizes a pulse, but that it can learn
a pulse \emph{distributionally adapted} to the noise model. This is the sense in which the present reproduction
most directly validates the main scientific message of the UFO framework: robustness against stochastic control
errors is not an accidental emergent feature of analog optimization, but a trainable property that can be
improved by embedding the appropriate perturbation model directly into the learning environment.

\subsection{Runtime sweep protocol}
\label{subsec:runtime_sweep_protocol}

To study hardware efficiency beyond the single-target setting, the analysis is extended from the representative gate
\(N(2.2,2.2,\pi/2)\) to the full one-parameter family
\begin{equation}
U_{\mathrm{target}}(\alpha)
=
N(\alpha,\alpha,\pi/2),
\qquad
\alpha \in [0,\pi].
\label{eq:runtime_target_family}
\end{equation}
The purpose of this subsection is therefore different from that of the robustness study in
section \ref{subsec:noise_model}--\ref{subsec:robustness_interpretation}. There, the central question was whether
stochastic training improves the stability of the learned controller under noisy perturbations. Here, by contrast,
the relevant question is whether direct analog control can realize different gates in the family
\(N(\alpha,\alpha,\pi/2)\) with runtimes that are competitive with, or even shorter than, those associated with
standard discrete gate synthesis. This makes the present sweep a family-level efficiency test rather than a
single-target robustness benchmark, in direct continuity with one of the two main empirical claims of the original
UFO paper \cite{Niu2019}. 
% :contentReference[oaicite:0]{index=0}

The primary runtime experiment is implemented as a \emph{curriculum TRPO sweep}. More precisely, for fixed
\(\gamma=\pi/2\), the controller is trained sequentially over increasing values of \(\alpha\), using the best
checkpoint obtained at one value of \(\alpha\) as the initialization for the next one. In the present code, the
parameter grid is generated from \(\alpha_{\min}=0\) to \(\alpha_{\max}=\pi\) with step size
\(\Delta\alpha=0.1\), while each phase is trained for at most \(100\) TRPO iterations, with
\(20{,}000\) episodes per batch, timestep \(\Delta t=2\,\mathrm{ns}\), maximum allowed duration
\(T_{\max}=600\,\mathrm{ns}\), and advance criterion based primarily on an evaluation-cost threshold of
\(0.15\). Operationally, this means that the runtime sweep is not performed by retraining each target completely
from scratch, but by exploiting continuity in target space: nearby gates in the family are assumed to admit
partially transferable control representations, so that the policy learned for one \(\alpha\) provides a useful
warm start for the next one. This choice is strongly aligned with the policy-based logic of the framework, since
TRPO is specifically designed to perform stable updates of continuous stochastic policies under a KL-constrained
trust-region step \cite{Schulman2015}. 
% :contentReference[oaicite:1]{index=1}

Alongside this curriculum-based RL sweep, a second experiment is carried out with the Adam baseline in order to
probe the same gate family without transfer learning. In that case, each target is optimized independently from
scratch, and the best runtime is selected through a horizon sweep over a discrete set of candidate durations. In
the implementation used here, the tested horizons are
\begin{equation}
T \in \{60,90,120,150,180,210,240\}\,\mathrm{ns},
\label{eq:adam_runtime_horizons}
\end{equation}
with \(400\) Adam iterations per horizon. The role of this second sweep is methodologically useful: it separates
the effect of analog controllability across the gate family from the additional benefit that may arise from
curriculum transfer. In other words, the Adam sweep provides a "from" family-level baseline for how
homogeneous or inhomogeneous the analog optimization problem already is, whereas the curriculum TRPO sweep tests
whether neighboring gates can be traversed more efficiently by reusing previously learned control structure.

The final ingredient of the protocol is the comparison with the optimal discrete gate-synthesis baseline discussed
earlier in Sec.~\ref{subsec:baselines_comparison_strategy}. Following the benchmark logic of the reference paper,
the relevant reference time is the estimated synthesized runtime
\begin{equation}
T_{\mathrm{syn}} \approx 215\,\mathrm{ns},
\label{eq:runtime_synthesis_baseline_recalled}
\end{equation}
obtained from the optimal decomposition of \(N(\alpha,\alpha,\gamma)\) into single-qubit rotations and entangling
gates, together with state-of-the-art gate-time estimates for those primitives \cite{Niu2019}. The runtime curves
reported below should therefore be interpreted against this fixed hardware-efficiency reference: for each
\(\alpha\), the central quantity of interest is whether the optimized analog controller lies below, near, or above
\(T_{\mathrm{syn}}\). This makes the sweep directly relevant to the broader physical question motivating the whole
section, namely whether direct analog control on the gmon can exploit the native interaction structure of the
device to outperform standard compiled implementations over a nontrivial family of two-qubit targets.

with \(\alpha\), the curriculum sweep exhibits a sequence of distinct regimes: an initial bootstrap region at very
small \(\alpha\), a broad central basin of comparatively fast solutions, a sharp minimum at
\(\alpha \approx \pi/2\), and a final degradation region as \(\alpha\) approaches \(\pi\). This already suggests
that the time required to implement the target gate is strongly dependent on how favorably the target aligns with
the native controllable dynamics of the gmon system, rather than being set by a uniform compilation overhead across
the whole family.

A first important point is that the \(\alpha=0\) phase should be interpreted separately from the rest of the
curve. In the present curriculum protocol, this is the only point that is not warm-started from a previous nearby
target, and therefore it acts as a genuine cold-start initialization of the whole sweep. Consistently with that
role, it is also the weakest point of the experiment: the final deterministic fidelity remains only
\(F \approx 0.558\), the runtime saturates at \(180\,\mathrm{ns}\), the leakage is still
\(4.8\times 10^{-3}\), and the corresponding \(\sigma=1\,\mathrm{MHz}\) average fidelity is only about \(0.613\).
For this reason, \(\alpha=0\) is operationally informative, but it should not be overinterpreted as representative
of the family-level runtime trend. The more meaningful structure of the curve emerges once the curriculum has
entered its transferable regime, starting already from \(\alpha \approx 0.157\).

Beyond this bootstrap point, the curve shows that the family is not uniformly difficult. A first reasonably good
region appears already for \(\alpha \approx 0.157\) and \(0.314\), where the controller reaches runtimes of
\(142\,\mathrm{ns}\) and \(140\,\mathrm{ns}\), with nominal fidelities above \(0.988\) and robust average
fidelities around \(0.951\) and \(0.953\), respectively. This indicates that the curriculum begins to exploit
cross-target continuity quite early. However, the next interval, roughly
\(\alpha \in [0.47,0.79]\), becomes harder again: the final runtime returns to the maximum observed value of
\(180\,\mathrm{ns}\), while the final fidelities drop to the \(0.948\)--\(0.954\) range and the
\(\sigma=1\,\mathrm{MHz}\) average fidelity falls to about \(0.906\)--\(0.918\). This is not merely a runtime
effect: the same region also exhibits visibly larger leakage and boundary-cost levels than the fast central basin,
which suggests that the controller is using less clean pulse shapes and paying a higher dynamical price to reach
the target.

The most striking feature of the figure is the central runtime valley around
\(\alpha \in [1.1,2.4]\), where the controller becomes substantially faster while maintaining very high quality.
In this interval, several targets are solved in the \(76\)--\(140\,\mathrm{ns}\) range, and the best part of the
basin lies roughly between \(\alpha \approx 1.884\) and \(\alpha \approx 2.355\), where the observed runtimes are
\(108\,\mathrm{ns}\), \(106\,\mathrm{ns}\), \(110\,\mathrm{ns}\), and \(118\,\mathrm{ns}\), all with nominal
fidelity above \(0.982\) and robust average fidelity at \(\sigma=1\,\mathrm{MHz}\) still around
\(0.954\)--\(0.959\). This is a genuinely important result: the runtime reduction is not obtained by sacrificing
accuracy, since the same part of the curve also remains in the high-fidelity and high-robustness regime. The curve
therefore indicates a broad family region in which fast analog implementations are not isolated accidents, but
repeatable solutions.

At the center of the family, the point \(\alpha=\pi/2\) stands out as an extreme minimum. Here the controller
terminates after only \(2\,\mathrm{ns}\), with essentially unit nominal fidelity
(\(F \approx 0.999999\)), negligible leakage (\(\sim 4\times 10^{-10}\)), extremely small boundary cost, and
average noisy fidelity still equal to \(0.9994\) at \(\sigma=1\,\mathrm{MHz}\). In addition, the corresponding
phase requires only a single curriculum iteration, making it not only the fastest gate in the sweep but also the
easiest one to learn. This is not a generic numerical fluctuation: it is a singular point in the empirical
landscape, and the sharpness of the dip strongly suggests a special dynamical compatibility between the target gate
at \(\alpha=\pi/2\) and the native interaction structure of the device. The physical interpretation of this point
will be discussed in detail in Paragraph~4, but already at the level of the curve it is clear that the runtime
landscape contains a privileged native-alignment point rather than a smooth continuum of equally hard targets.

After this central minimum, the runtime rises again but not immediately catastrophically. The interval
\(\alpha \in [1.727,2.669]\) still forms a favorable region: runtimes remain between \(124\,\mathrm{ns}\) and
\(144\,\mathrm{ns}\), fidelities stay in the \(0.982\)--\(0.992\) range, and robustness remains high. Only when
the sweep enters the final high-\(\alpha\) edge, roughly \(\alpha \gtrsim 2.826\), does the curve degrade more
clearly. In that last region, the runtime saturates again at \(180\,\mathrm{ns}\), while nominal fidelity falls
to \(0.922\)--\(0.940\) and robust average fidelity to \(0.887\)--\(0.902\). This suggests that the favorable
transfer structure exploited by the curriculum over the central basin becomes progressively weaker as the target is
pushed toward the far end of the family.

A further nontrivial insight emerges when the final runtime is compared with the
\emph{minimum-cost time} reached within the same phase. For many of the well-converged points, the two coincide,
which indicates that the controller not only discovers a good time scale but also stabilizes it as the final
solution. However, in several difficult regions this coincidence breaks down. For example, around
\(\alpha \approx 0.471\) and \(0.628\), the final runtime is \(180\,\mathrm{ns}\) even though the lowest-cost
intermediate trajectory is reached already around \(142\)--\(146\,\mathrm{ns}\); similarly, near
\(\alpha \approx 2.826\), \(2.983\), and \(\pi\), the minimum-cost times lie around \(152\)--\(158\,\mathrm{ns}\)
while the final rollout still ends at \(180\,\mathrm{ns}\). This indicates that in the hard regions the agent can
transiently visit faster candidate solutions, but does not consistently stabilize them under the full optimization
criterion. In other words, the difficult parts of the runtime curve are not simply regions where short solutions do
not exist at all; rather, they are regions where short solutions appear to be less stable, less robust, or harder
to preserve throughout training.

The same reading is supported by the auxiliary diagnostics shown in
Figs.~\ref{fig:runtime_aux_performance_alpha} and \ref{fig:runtime_aux_constraints_alpha}. The fastest phases are
also the ones with the cleanest constraint profile: lower leakage, lower boundary cost, and, in general, better
agreement between nominal and noisy fidelity. Conversely, the slowest phases are precisely those where leakage and
boundary penalties rise again. This coupling between runtime and constraint satisfaction is crucial, because it
shows that the runtime curve is not an isolated geometric observable; it is embedded in the same control trade-off
already discussed for the single-target case. Short runtimes are achieved when the controller can simultaneously
realize the target, suppress leakage, and keep the pulse endpoints under control. Long runtimes reappear when one
or more of these constraints become harder to satisfy at the same time.

Overall, the runtime curve therefore conveys a richer message than a simple pointwise comparison of gate durations.
It shows that the family \(N(\alpha,\alpha,\pi/2)\) contains a broad favorable analog-control basin, centered
around a particularly privileged point at \(\alpha=\pi/2\), but also flanked by regions where the controller
either needs longer trajectories or fails to stabilize the shorter ones it transiently discovers. This makes the
curve especially informative for the rest of the subsection: it is not only a record of achieved runtimes, but a
map of where the analog-control landscape is intrinsically easy, where it is fragile, and where the curriculum
transfer mechanism is most and least effective.

\subsection{Comparison with gate synthesis}
\label{subsec:comparison_gate_synthesis}

A central question raised by the runtime sweep is whether the analog-control solutions found by the curriculum-trained
TRPO agent are merely competitive with standard discrete compilation, or whether they actually outperform it in a
meaningful way. To answer this, the natural reference is the optimal gate-synthesis decomposition of the family
\(N(\alpha,\alpha,\gamma)\), discussed in the original UFO paper on the basis of the Vatan--Williams two-qubit
synthesis result \cite{VatanWilliams2004,Niu2019}. In that benchmark, the target unitary is decomposed into a
minimal-depth circuit over arbitrary one-qubit rotations and entangling gates, yielding an estimated runtime of
approximately \(215\,\mathrm{ns}\) for the \(N(\alpha,\alpha,\gamma)\) family under the hardware timing assumptions
adopted in \cite{Niu2019}. Although this is not a pulse-level optimization in the exact same analog-control space
used in the present work, it is still the correct compilation baseline for assessing whether direct continuous
control provides a genuine temporal advantage over standard discrete synthesis.

Under that reference, the present runtime sweep shows that the analog-control strategy is not just occasionally
better: across essentially the entire nontrivial family \(N(\alpha,\alpha,\pi/2)\), it remains below the
gate-synthesis benchmark. Even the difficult edge points of the sweep, where the learned controller saturates at
\(180\,\mathrm{ns}\), still remain faster than the synthesis estimate by about \(35\,\mathrm{ns}\), i.e. by roughly
\(16\%\). This already matters, because those are precisely the hardest regions of the landscape, where nominal
fidelity and noisy fidelity both deteriorate and where the curriculum no longer finds the clean short-time solutions
available in the central basin. In other words, the analog approach preserves a runtime advantage even where the
optimization problem becomes visibly more difficult.

The advantage becomes much more substantial in the broad central part of the family. Around
\(\alpha \in [1.73,2.36]\), the final learned runtimes lie between \(106\,\mathrm{ns}\) and \(124\,\mathrm{ns}\),
which corresponds to a reduction of about \(42\%\) to \(51\%\) relative to the \(215\,\mathrm{ns}\) gate-synthesis
reference. In this regime, the speedup is therefore not marginal but systematic. Importantly, this reduction does
not come with a collapse in control quality: the same points remain in the high-fidelity regime
(\(F \approx 0.982\)--\(0.986\)) and maintain robust average fidelity near \(0.955\)--\(0.959\) at
\(\sigma=1\,\mathrm{MHz}\). The comparison is therefore qualitatively stronger than a bare runtime statement:
for a substantial subset of the gate family, analog control is simultaneously faster, accurate, and reasonably
robust, rather than merely trading correctness for speed.

An even stronger discrepancy appears at selected favorable points of the landscape. For instance, at
\(\alpha \approx 1.10\) the learned runtime drops to \(76\,\mathrm{ns}\), corresponding to a speedup of almost a
factor \(3\) over the synthesis baseline, while still retaining nominal fidelity above \(0.98\) and robust average
fidelity above \(0.96\). The singular point \(\alpha=\pi/2\) is more extreme still: the controller terminates after
\(2\,\mathrm{ns}\), with essentially unit nominal fidelity and negligible leakage. This value should not be treated
as representative of the entire family, because it reflects a very special alignment between target and native
Hamiltonian structure; nevertheless, it is conceptually important. It demonstrates that, at least for some targets,
the compilation overhead implicit in discrete gate synthesis is not a fundamental lower bound on execution time.
Rather, it is an artifact of expressing the target through a universal discrete gate alphabet instead of exploiting
the analog structure of the hardware directly.

This distinction is crucial for interpreting the result correctly. The discrete synthesis benchmark is "optimal"
within a circuit model based on a prescribed elementary gate library, whereas the analog controller optimizes
directly in the continuous Hamiltonian-control space. The two are therefore solving related but not identical
optimization problems. For this reason, the present comparison should not be read as a statement that the analog
controller has beaten an absolute quantum speed limit. What it does show, however, is that once the hardware admits
a rich native interaction and sufficiently flexible pulse shaping, the shortest experimentally meaningful
implementation of a target gate need not coincide with its shortest decomposition over a discrete universal gate set.
From the point of view of near-term hardware efficiency, this is precisely the relevant notion of improvement.

The comparison also clarifies where the gain is modest and where it is dramatic. In the hard regions near the two
edges of the sweep, the analog controller is only moderately faster than gate synthesis, and the runtime gap narrows
precisely when leakage, boundary penalties, and robustness degradation become more pronounced. In the central basin,
by contrast, the gap widens sharply. This indicates that the benefit of analog control is strongly target-dependent:
it is largest when the target gate can be realized in a way that is geometrically and dynamically aligned with the
native couplings of the physical device, and smallest when extra control resources are needed to compensate for a
less favorable target structure. Thus, the runtime comparison is not only a numerical benchmark, but also a
diagnostic of how much compilation overhead can be removed by working directly at the Hamiltonian level.

Overall, the results support the same qualitative conclusion reported in the original UFO work, but in a more
fine-grained family-resolved form. The analog-control solutions found here are broadly faster than the standard
gate-synthesis reference, often by a large margin, and the strongest gains occur exactly in the regions where the
control landscape appears most compatible with the underlying gmon interaction. This confirms that the runtime
advantage of analog control is real, not anecdotal: for this gate family, direct pulse-level optimization can indeed
surpass standard discrete synthesis as a practically relevant notion of implementation efficiency.

\subsection{Physical interpretation}
\label{subsec:physical_interpretation}

The runtime landscape observed in Fig.~\ref{fig:runtime_vs_alpha} is physically meaningful, and should not be read as a purely numerical peculiarity of the optimizer. The key point is that the target family considered here,
\(N(\alpha,\alpha,\pi/2)\), is not arbitrary with respect to the underlying gmon Hamiltonian. As discussed in the system-model section, the native entangling interaction is generated primarily by the exchange-like coupling term
\(g(t)\,(\hat a_2^\dagger \hat a_1+\hat a_1^\dagger \hat a_2)\), whose effective action on the computational subspace is naturally aligned with the
\(XX+YY\) sector of the two-qubit interaction. Consequently, when the desired gate requires a balance of nonlocal content that can be synthesized largely along this native direction, the controller does not need to "fight" the hardware: it can exploit the intrinsic entangling channel directly, while using the remaining controls mainly to shape phases, suppress leakage, and satisfy the boundary constraints. In precisely this regime, shorter runtimes are not only empirically observed, but physically expected. This is fully consistent with the original motivation of the UFO framework, which emphasizes that analog control becomes especially advantageous when the target gate is geometrically close to the entangling structure naturally generated by the device Hamiltonian \cite{Niu2019}.

This interpretation also explains why the best-performing region of the sweep is concentrated in the middle of the
\(\alpha\)-interval rather than at its boundaries. In the central basin, the optimizer can realize the required nonlocal action with relatively modest auxiliary control effort: the pulse does not need large corrective excursions in the local channels, the boundary term remains moderate, and the TSWT leakage proxy stays under control. From a dynamical viewpoint, the gate can be implemented by letting the system evolve along a direction that is already available in hardware, rather than by repeatedly compensating unwanted motion through more elaborate control shaping. The shorter runtimes found in this region are therefore plausibly associated with smoother and more "natural" trajectories in control space. This interpretation is reinforced by the fact that the same \(\alpha\)-window also displays strong nominal fidelities and good noisy fidelities at \(\sigma=1\,\mathrm{MHz}\): the region that is faster is also the region that is easier to control reliably. In that sense, temporal efficiency and robustness are not independent observations here, but two manifestations of the same underlying alignment between target geometry and native device physics.

By contrast, the growth of runtime toward the two edges of the family is also physically interpretable. As \(\alpha\) moves away from the favorable central window, the desired gate becomes less directly compatible with the entangling evolution that the gmon hardware provides most naturally. The controller must then rely more heavily on additional detuning and microwave components to steer the dynamics toward the target while simultaneously suppressing population transfer outside the computational subspace. This extra compensation effort has two immediate consequences. First, it tends to lengthen the control sequence, because more time is needed to accumulate the correct effective nonlocal action without violating the leakage and boundary penalties. Second, it makes the solution more fragile, since longer and more structured pulses expose the system for a greater duration to both coherent modeling imperfections and stochastic perturbations. The qualitative correlation observed across the sweep (higher runtimes, slightly worse robust fidelities, and reduced margin to the gate-synthesis baseline near the edges) is therefore exactly what one would expect if those target gates were dynamically less aligned with the native Hamiltonian.

The special behavior near \(\alpha=\pi/2\) should be interpreted in the same spirit, but with additional caution. The extremely short runtime found there suggests that this point may correspond to a particularly favorable symmetry or local-equivalence configuration within the gate family, so that the desired transformation can be reached with very little pulse duration once the controller is placed in the appropriate dynamical sector. This does not mean that \(2\,\mathrm{ns}\) should be taken as a generic realizability statement for arbitrary two-qubit gates, nor that the analog controller has saturated a universal quantum speed limit. Rather, it indicates that within a restricted family and on a hardware platform with a strongly structured native interaction, isolated gates may exist whose implementation cost collapses because the target is exceptionally well matched to the underlying entangling generator. For the purposes of the present thesis, the important conclusion is therefore not the extremal value itself, but the broader structural pattern around it: the runtime minima occur where physical compatibility is highest.

Taken together, these observations support the main conceptual message of this subsection. The runtime advantage of analog control over discrete synthesis is largest not in a random subset of the target family, but precisely where the target gate is best aligned with the natural interaction channels of the superconducting device. This makes the result much stronger than a purely algorithmic benchmark. It shows that the improvement is rooted in the physics of the platform: direct pulse-level optimization can remove part of the compilation overhead associated with universal gate decomposition because it exploits the Hamiltonian that the hardware already implements natively. Conversely, when that alignment deteriorates, the runtime advantage shrinks, the controller must work harder to enforce fidelity and leakage constraints, and the solutions become less efficient. The runtime sweep therefore provides a concrete dynamical interpretation of why analog control can outperform standard gate synthesis on this family: it is not merely optimizing faster, but optimizing along the natural directions of the device itself.
