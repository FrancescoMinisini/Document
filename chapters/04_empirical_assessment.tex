\subsection{Compared methods}
Within this common framework, four optimization and comparison schemes are considered. 
\begin{itemize}
    \item The first is a direct gradient-based baseline, implemented through the Adam optimizer, which searches directly over the filtered control trajectory for a fixed gate horizon. 
    \item The second is a policy-gradient reinforcement-learning controller trained in the nominal environment, that is, without stochastic perturbations applied during training. 
    \item The third is the noise-optimized TRPO controller, which is trained in a stochastic environment where Gaussian perturbations are injected into the control amplitudes and into the anharmonicity-related parameter entering the Hamiltonian. 
    \item The fourth comparison is not a pulse-level optimizer but a circuit-level reference, namely the optimal gate-synthesis baseline used to benchmark runtime on the family \(N(\alpha,\alpha,\gamma)\). 
\end{itemize}
These four methods play different methodological roles. The Adam baseline tests whether a standard first-order optimization of the raw control variables is sufficient to obtain high-quality pulses. The nominal TRPO controller tests whether reinforcement learning is already competitive in the idealized deterministic setting. The noise-optimized TRPO controller tests whether robustness can be learned directly from stochastic training. Finally, the gate-synthesis baseline tests whether direct analog control offers a runtime advantage over a discrete compiled implementation of the same target gate. Taken together, these methods define a comparison protocol that is not centered on a single optimizer, but on a set of scientifically distinct questions concerning accuracy, robustness, and hardware efficiency.

\subsection{Metrics}
The performance of each control method is evaluated through a set of complementary metrics designed to probe different aspects of the control problem. At the most basic level, the implemented gate is assessed through its fidelity with respect to the target unitary. For a final physical propagator \(U(T)\), projected onto the computational subspace, the gate fidelity is computed as
\begin{equation}
F_{\mathrm{gate}}
=
\frac{1}{16}
\left|
\mathrm{Tr}\!\left(
U_{\mathrm{proj}}^{\dagger}(T)\,U_{\mathrm{target}}
\right)
\right|^2,
\label{eq:validation_gate_fidelity}
\end{equation}
where \(U_{\mathrm{target}}=N(\alpha,\alpha,\gamma)\). This quantity is reported together with the total UFO cost, which combines fidelity, leakage, boundary penalties, and runtime into a single scalar objective. Since the framework is explicitly multilevel, nominal performance is not characterized by fidelity alone. For every optimized control plan, the simulation also returns the TSWT-based leakage estimate \(L_{\mathrm{tot}}\), its decomposition into boundary and integral contributions, the boundary penalty term, and the final runtime in nanoseconds. These quantities allow one to distinguish between a control that is merely accurate on the logical subspace and one that is also physically well behaved in the full truncated Hilbert space. In addition to nominal performance, robustness is evaluated statistically under sampled noisy control realizations. Given a control plan \(\tilde{\bm{c}}\) and a Gaussian noise level \(\sigma_{\mathrm{noise}}\), the key robustness diagnostics are the average fidelity
\begin{equation}
\overline{F}
=
\mathbb{E}_{\delta\tilde{\bm{c}}}
\Bigl[
F\bigl(U(\tilde{\bm{c}}+\delta\tilde{\bm{c}})\bigr)
\Bigr],
\label{eq:validation_average_fidelity}
\end{equation}
the corresponding average gate fidelity, and the fidelity variance
\begin{equation}
\mathrm{Var}(F)
=
\mathbb{E}_{\delta\tilde{\bm{c}}}
\left[
\bigl(F-\overline{F}\bigr)^2
\right].
\label{eq:validation_fidelity_variance}
\end{equation}
These metrics are essential because they distinguish mean robustness from shot-to-shot stability: a control scheme with high average fidelity but large variance remains experimentally fragile. Finally, for the runtime study, the relevant metric is the total control duration required to realize a target gate, compared against the optimal discrete gate-synthesis baseline. The resulting evaluation protocol is therefore explicitly multidimensional: it measures logical accuracy, leakage suppression, robustness under stochastic perturbations, and hardware efficiency rather than relying on a single scalar score.

\subsection{Evaluation protocol}
All experiments follow a common procedural logic. For a given target gate \(N(\alpha,\alpha,\gamma)\), a control method is first trained or optimized under its own learning rule until either a prescribed iteration budget is exhausted or a termination criterion on the UFO cost is satisfied. The resulting best control plan is then stored and re-evaluated in a deterministic nominal setting in order to extract its final cost, fidelity, leakage, and runtime under identical physical assumptions. For robustness studies, the same saved control plan is subsequently subjected to repeated noisy rollouts, in which Gaussian perturbations are independently sampled and injected into the control channels and into the anharmonicity-related parameter of the Hamiltonian. This Monte Carlo evaluation yields the empirical estimates entering Eqs.~\eqref{eq:validation_average_fidelity} and \eqref{eq:validation_fidelity_variance}. In the runtime study, by contrast, the focus is not on statistical robustness but on how the optimized gate duration varies across the target family, especially along the subfamily \(N(\alpha,\alpha,\pi/2)\), where analog control can be meaningfully compared with optimal gate synthesis. The same timestep, control bounds, bandwidth assumptions, and physical truncation are retained throughout, so that differences between methods can be attributed to the optimization strategy rather than to inconsistencies in the simulation model. This common evaluation protocol is particularly important for the interpretation of the results: it ensures that the subsequent comparisons are methodologically fair, while also making the distinction between nominal accuracy, noisy robustness, and runtime efficiency completely explicit.

The choice of target gates follows the logic introduced in the previous section. For the single-target experiments, a representative gate of the form \(N(2.2,2.2,\pi/2)\) is particularly useful because it is nontrivial, lies in the physically relevant family studied by the original paper, and is also the gate used in the paper’s robustness comparison under Gaussian control noise. More broadly, the family \(N(\alpha,\alpha,\gamma)\) remains the natural benchmark because it contains gates of direct relevance to quantum simulation and fermionic algorithms while still being sufficiently aligned with the native interaction structure of the gmon Hamiltonian to make the analog-versus-synthesis comparison meaningful. This is why the same family is retained throughout the thesis both for validation and for later extensions.

Among the RL-specific hyperparameters, the initialization of the stochastic policy deserves separate discussion. In the present implementation, the policy outputs a Gaussian distribution over the raw control actions, followed by a \(\tanh\) squashing map to enforce bounded actions. An initial value \(\texttt{init\_log\_std}=-1.0\) corresponds to an initial standard deviation of approximately \(e^{-1}\approx 0.37\) in the raw action space, which provides non-negligible exploration without immediately saturating the squashing nonlinearity. In practice, this proved to be a good compromise: larger initial values made early exploration too aggressive and frequently pushed actions toward the saturation regime, while smaller values reduced exploration too strongly and slowed discovery of useful control trajectories. This parameter is not specified explicitly in the original paper and should therefore be regarded as an implementation-level design decision rather than a paper-level constant.

A second key RL hyperparameter is the TRPO trust-region radius \(\texttt{max\_kl}\). In TRPO, the KL-divergence constraint controls how far the updated policy is allowed to move from the current one in a single optimization step. A value that is too large may produce unstable updates and overshooting, especially in noisy environments and in nonconvex control landscapes; a value that is too small, on the other hand, yields very conservative learning and may require many more iterations to achieve a comparable solution. For the full runs, a nominal choice \(\texttt{max\_kl}=0.01\) was retained, since this lies close to the standard TRPO setting and provided a good empirical compromise between update speed and update reliability. The related parameters \(\gamma_{\mathrm{RL}}=0.99\), \(\lambda=0.97\), damping \(=0.1\), and 10 conjugate-gradient iterations were kept at stable standard values, as their main role is to regularize the on-policy update rather than to define the physics of the control problem itself.

The temporal discretization also involves a nontrivial trade-off. A fine mesh, such as \(\Delta t=2\,\mathrm{ns}\), allows more detailed pulse shaping and better representation of rapidly varying controls, but it also increases the episode length and therefore the dimensionality of the sequential decision problem. Likewise, a larger maximum duration \(T_{\max}\) expands the reachable control space, but at the cost of substantially longer rollouts and higher training expense. For the main experiments, the pair \((\Delta t,T_{\max})=(2\,\mathrm{ns},180\,\mathrm{ns})\) was adopted as a practical compromise: the mesh remains fine enough to resolve meaningful pulse structure, while the total time horizon stays within a regime that is both computationally manageable and physically interesting for fast two-qubit gate synthesis. Coarser choices, such as \(\Delta t=10\,\mathrm{ns}\), are still useful for rapid exploratory sweeps, but they are less suitable for the final accuracy-oriented runs. This trade-off is closely connected to the broader speed--leakage problem in superconducting-qubit control: finer and longer horizons allow more flexibility, but overly aggressive short-time control can easily increase leakage, while overly long horizons weaken the main hardware-efficiency motivation of the method.

(\chi,\beta,\mu,\kappa)=(10.0,10.0,0.2,0.1),
\label{eq:chosen_ufo_weights}
\end{equation}
which reflect a balanced emphasis on logical accuracy and leakage suppression, together with weaker penalties on boundary mismatch and runtime. This choice is not arbitrary: the paper reports these values as the result of a grid search over the cost parameters and uses them uniformly across the target-gate family. In the present thesis, these values are adopted as the default benchmark setting, precisely because they allow the later analysis to distinguish between effects due to the optimizer and effects due to reweighting of the physical objective. A more systematic sweep of the UFO weights is postponed to the later ablation study, where the sensitivity of the method to the cost scalarization can be studied explicitly.

The stopping threshold, or terminal cost, deserves a particularly explicit justification. In the codebase, this parameter determines whether an episode is terminated early once the running UFO cost falls below a prescribed value. A relatively loose threshold, for example around \(0.4\), makes early stopping easier to trigger and can shorten training time substantially, but it also risks accepting pulse trajectories that are only superficially satisfactory, especially if their low cost is achieved through short runtime rather than genuinely high fidelity and low leakage. By contrast, a stringent threshold such as \(0.15\) is physically more meaningful but much harder to satisfy within a fixed time horizon. In the present work, the stricter value \(0.15\) was retained for the main runs, precisely because it avoids prematurely truncating the search toward easy but uninteresting controls. In practice, this threshold is rarely reached during noisy training. For that reason, a more informative quantity for analysis is often the \emph{minimum cost attained during training} together with the corresponding timestamp or iteration index, since this reveals how low the optimizer can actually drive the cost even when the formal termination criterion is not triggered.

The robustness-evaluation parameters are chosen to remain as close as possible to the original paper. During training, Gaussian amplitude fluctuations with standard deviation \(1\,\mathrm{MHz}\) are injected into the relevant control channels, corresponding to the paper’s nominal noisy-training environment. During evaluation, the noise amplitude is swept over the interval \(0.1\) to \(3.5\,\mathrm{MHz}\), and each point is estimated from 60 noisy realizations. This setup reflects two complementary goals: first, to retain direct comparability with the original benchmark; second, to avoid drawing conclusions from a single arbitrary noise amplitude. The paper explicitly motivates \(1\,\mathrm{MHz}\) as a reasonable experimentally relevant scale, while also emphasizing that the true noise level is generally uncertain and may drift in time. The use of 60 samples per evaluation point is therefore a compromise between statistical stability and total computational cost.

The remaining implementation choices mainly reflect computational trade-offs. The original paper updates the policy after batches of 20,000 episodes, which is large but consistent with the data-hungry nature of on-policy methods. In the present implementation, full comparison runs may use either 20,000 episodes per batch, to stay as close as possible to the original setup, or 10,000 episodes per batch, as a lighter but still stable alternative when computational budget is more limited. Similarly, around 100 iterations are used for full runs, while much shorter runs, such as 5 iterations, are sufficient for parameter-search sweeps and fast sanity checks. The main trade-off is straightforward: larger batches and more iterations improve the reliability of policy updates and the eventual convergence quality, but the wall-time cost grows very quickly because each batch requires a large number of multilevel Schrödinger rollouts. In addition, the codebase contains several secondary but non-negligible parameters, such as the filter bandwidth \(10\,\mathrm{MHz}\), the maximum control amplitude \(20\,\mathrm{MHz}\), the baseline anharmonicity \(-200\,\mathrm{MHz}\), the Adam learning rate and horizon set for the baseline optimizer, and the parallel rollout configuration. These parameters are not the central scientific objects of the thesis, but they matter in practice because they determine the smoothness of the controls, the degree of physical realism, and the total computational burden of the experiments.

Finally, the neural-network architecture was kept at the paper’s nominal configuration, namely two fully connected three-layer networks of width \(64\), \(32\), and \(32\) for the policy and value functions, respectively. This preserves direct comparability with the reference implementation and avoids conflating architectural changes with control-theoretic conclusions. At the same time, it is well known from the RL literature that network architecture can materially affect performance, especially in on-policy continuous-control settings. For that reason, architectural sensitivity should be regarded as an important follow-up question rather than as fully settled here.

\section{Single-target control optimization}
\label{sec:single_target_optimization}
\subsection{Why this target is representative}
\label{subsec:why_this_target_is_representative}

As a representative nominal benchmark, this study first focuses on the target gate
\begin{equation}
U_{\mathrm{target}}
=
N(2.2,2.2,\pi/2),
\label{eq:representative_target_gate}
\end{equation}
which belongs to the two-qubit family \(N(\alpha,\alpha,\gamma)\) introduced in the previous section. This choice is motivated by three considerations. 
\begin{itemize}
    \item First, it is a genuinely nontrivial target: it is neither close to the identity nor reducible to a particularly simple limiting case, and therefore provides a meaningful test of the optimizer's ability to coordinate entangling dynamics, local control, and leakage suppression within the full multilevel model. 
    \item Second, it is directly aligned with the original benchmark logic of the reference paper, which uses the same gate family as the main testbed for robustness and runtime analyses \cite{Niu2019}. For this reason, selecting a target of the form \(N(\alpha,\alpha,\pi/2)\) allows the present study to remain as close as possible to the original physical and algorithmic setting while still isolating one concrete case for detailed discussion. 
    \item Third, fixing a single representative gate provides a clean comparison between optimization strategies before moving to broader parameter sweeps: by holding the target fixed, differences in final fidelity, leakage, runtime, and training behavior can be attributed more directly to the optimizer itself rather than to variations in target-gate difficulty across the family. 
\end{itemize}
In this sense, Eq.~\eqref{eq:representative_target_gate} serves as the natural entry point for the empirical analysis, since it is sufficiently rich to be physically informative while remaining simple enough to support a transparent method-by-method comparison.
\subsection{Adam baseline results.}
As a strong non-RL nominal baseline, the representative target gate \(N(2.2,2.2,\pi/2)\) was optimized by direct pulse search with Adam over a sweep of candidate horizons from \(40\) to \(180\,\mathrm{ns}\), using the same timestep \(\Delta t=2\,\mathrm{ns}\), the same UFO weights \((\chi,\beta,\mu,\kappa)=(10,10,0.2,0.1)\), and no stochastic perturbations during training. This matched setup is important for the later comparison with TRPO, because the physical model, discretization, and scalarized objective are kept fixed and only the optimization strategy is changed. In this sense, Adam plays the role of a conventional open-loop pulse-optimization baseline, conceptually close to standard gradient-based quantum-control workflows \cite{Khaneja2005,Kingma2015}.

The horizon sweep reveals a clear operational threshold around \(70\,\mathrm{ns}\). For very short gates, Adam is unable to satisfy the competing requirements of high fidelity, low leakage, and small boundary penalty simultaneously. At \(40\,\mathrm{ns}\), the optimized pulse remains far from acceptable, with nominal cost \(C=3.142\), fidelity \(F=0.7327\), leakage estimate \(L_{\mathrm{tot}}=3.76\times 10^{-4}\), and a large boundary contribution of \(3.99\times10^{-1}\). Even at \(60\,\mathrm{ns}\), although the fidelity has already risen to \(F=0.9847\), the total cost is still \(C=0.3413\), largely because the boundary contribution remains substantial \((8.51\times10^{-2})\) and the leakage estimate is still \(3.24\times10^{-4}\). Moving from \(60\) to \(70\,\mathrm{ns}\) produces the sharpest qualitative improvement of the whole sweep: the nominal fidelity jumps to \(F=0.999385\), the leakage estimate drops to \(1.98\times10^{-4}\), and the boundary contribution collapses by roughly two orders of magnitude, to \(8.31\times10^{-4}\). The best Adam solution is therefore obtained at \(70\,\mathrm{ns}\), with objective cost \(C=0.12563\), fidelity \(F=0.999385\), leakage estimate \(L_{\mathrm{tot}}=1.98\times10^{-4}\), boundary contribution \(8.31\times10^{-4}\), and weighted runtime contribution \(0.1167\). Moreover, the optimizer cost and the independently re-evaluated nominal cost coincide up to numerical precision, which supports the internal consistency of the optimization and evaluation pipeline.

This behavior is directly consistent with the discussion of the speed--leakage tradeoff in Section~\ref{subsec:speed_leakage_tradeoff}. Below a certain runtime, the control does not have enough temporal freedom to realize the target gate while also turning the pulses on and off smoothly and keeping leakage under control. In this sense, the sharp transition between \(60\) and \(70\,\mathrm{ns}\) can be interpreted as a practical minimum-time threshold under the imposed physical constraints, rather than as a purely numerical accident. It is also noteworthy that the dominant gain in this regime is not only a fidelity improvement, but also a dramatic reduction in the boundary penalty, which indicates that a substantial part of the difficulty of the short-horizon problem comes from satisfying the turn-on/turn-off conditions built into the UFO objective.

A second important pattern is that \(70\,\mathrm{ns}\) is \emph{not} the point of highest nominal fidelity, but the point of lowest \emph{total UFO cost}. For longer horizons, the fidelity keeps improving monotonically, reaching \(F=0.999923\) at \(180\,\mathrm{ns}\), while the leakage estimate decreases to \(1.12\times10^{-5}\). However, these gains are too small to compensate for the linear growth of the runtime term in the scalarized objective. The Adam sweep therefore provides a particularly transparent illustration of the logic of the UFO cost introduced in Section~\ref{subsec:ufo_cost_function}: the preferred solution is not the slowest and cleanest pulse in absolute terms, but the pulse that best balances accuracy, leakage suppression, boundary regularity, and execution time. In this sense, the optimum at \(70\,\mathrm{ns}\) should be interpreted as a \emph{practical multi-objective optimum} rather than as a pure fidelity optimum. The monotonic reduction of leakage with increasing horizon is also physically consistent with the standard picture for weakly anharmonic superconducting qubits, where longer and smoother pulses reduce spectral spillover into higher levels and therefore suppress leakage \cite{Motzoi2009,Gambetta2011}.

From the broader perspective of the thesis, the Adam baseline already establishes two important points. 
\begin{itemize}  
\item First, direct analog control itself is highly competitive in the nominal setting: the best Adam pulse at \(70\,\mathrm{ns}\) is approximately a factor of \(3.1\) shorter than the \(215\,\mathrm{ns}\) optimal gate-synthesis estimate reported for the same gate family in Ref.~\cite{Niu2019}. This strongly supports the hardware-efficiency motivation discussed in Sections~\ref{subsec:intro_analog_vs_synthesis} and~\ref{subsec:baselines_comparison_strategy}, and suggests that a substantial part of the runtime advantage comes from the analog-control formulation itself rather than from reinforcement learning alone. 
\item Second, nominal optimality does not imply robustness. 
When the best Adam plan is evaluated a posteriori under Gaussian amplitude noise with \(\sigma=1\,\mathrm{MHz}\), its average fidelity drops to \(\overline{F}=0.97956\), with average gate fidelity \(0.97445\) and fidelity variance \(7.88\times10^{-5}\). 
This gap between excellent nominal performance and noticeably degraded noisy performance is precisely the kind of behavior that motivates the robust RL framework studied in this thesis: a nominal optimizer can already find very fast and accurate analog solutions, but without explicit exposure to stochastic perturbations during training, those solutions need not be robust in the sense emphasized by the original UFO framework \cite{Niu2019}. 
\end{itemize}

The Adam results therefore provide a meaningful and nontrivial baseline: they confirm that the nominal control problem is solvable at high quality, while simultaneously showing why robustness must be treated as a central objective rather than as a post hoc diagnostic.

\subsection{Nominal TRPO results}
I next considered the same representative target gate \(N(2.2,2.2,\pi/2)\) within the reinforcement-learning formulation of the UFO framework, but in a \emph{nominal} training regime, that is, without stochastic control perturbations injected during training. In this experiment, the controller was trained with trust-region policy optimization (TRPO) \cite{Schulman2015}, using the same physical target and the same underlying gmon simulation model as in the Adam baseline, with timestep \(\Delta t = 2\,\mathrm{ns}\), maximum episode duration \(180\,\mathrm{ns}\), and the same UFO weights \((\chi,\beta,\mu,\kappa)=(10,10,0.2,0.1)\). The policy and value networks retained the \(64\text{--}32\text{--}32\) fully connected architecture used in the original UFO paper, while the main TRPO hyperparameters were set to \(\texttt{max\_kl}=0.01\), \(\gamma_{\mathrm{RL}}=0.99\), \(\lambda=0.97\), and \(\texttt{init\_log\_std}=-0.5\). Training was performed for \(100\) policy iterations with batches of \(10^4\) episodes per iteration, using 8 rollout workers.

Under this nominal TRPO setting, the best deterministic evaluation achieved
\begin{equation}
C_{\mathrm{TRPO}}^{\star}=0.3676,
\end{equation}
with corresponding fidelity
\begin{equation}
F_{\mathrm{nom}}=0.99155,
\end{equation}
leakage estimate
\begin{equation}
L_{\mathrm{tot}}=4.59\times 10^{-4},
\end{equation}
and gate time
\begin{equation}
T=102\,\mathrm{ns}.
\end{equation}
These values show that the RL agent is able to learn a fully viable analog control policy for the target gate in the nominal environment: the resulting pulse remains in the high-fidelity regime, stays well below the synthesized-gate reference time of about \(215\,\mathrm{ns}\) discussed in Ref.~\cite{Niu2019}, and therefore already exhibits nontrivial hardware efficiency. In particular, the achieved duration of \(102\,\mathrm{ns}\) is still roughly a factor of two shorter than the circuit-synthesis reference, which indicates that direct analog control remains competitive at the runtime level even in the absence of noisy-environment training.

At the same time, the comparison with the direct Adam baseline is revealing. On this single fixed target, the nominal TRPO controller does \emph{not} outperform the direct pulse optimizer: the Adam baseline reached lower cost, higher fidelity, lower leakage, and a shorter runtime, with
\begin{equation}
C_{\mathrm{Adam}}^{\star}=0.1256,\qquad
F_{\mathrm{Adam}}=0.999385,\qquad
L_{\mathrm{Adam}}=1.98\times 10^{-4},\qquad
T_{\mathrm{Adam}}=70\,\mathrm{ns}.
\end{equation}
This difference is substantial enough that it should be interpreted, rather than ignored. In the present nominal single-target setting, the direct gradient-based optimizer appears to be the stronger method for extracting a high-quality pulse from a differentiable simulator. This is not particularly surprising from the viewpoint of the broader quantum optimal-control literature: when the target is fixed, the simulator is known, and the objective is smooth and directly evaluable, strong direct optimizers such as GRAPE-style methods or related gradient-based pulse searches remain very hard to beat on purely nominal performance \cite{Khaneja2005,deFouquieres2011,Machnes2018}.

However, this outcome does not undermine the relevance of the RL formulation; rather, it clarifies what should and should not be claimed at this stage. First, the present run should be interpreted as a \emph{nominal feasibility benchmark}, not as the sharpest possible TRPO optimum, since the stopping rule used here was intentionally looser than in the Adam baseline: the nominal TRPO training employed a termination threshold of \(0.4\), whereas the Adam sweep used \(0.15\). This likely biases the RL agent toward finding an acceptable pulse earlier, rather than continuing to refine the solution toward the lowest attainable nominal cost. Second, and more importantly, the original motivation of the UFO framework is not merely to solve a single deterministic fixed-target optimization problem, but to learn policies that remain meaningful once leakage, runtime, and stochastic control errors are taken into account jointly \cite{Niu2019}. In that broader perspective, the fact that TRPO is already able to produce a \(102\,\mathrm{ns}\) high-fidelity controller in the nominal setting is best viewed as a proof of capability rather than as the final scientific endpoint.

The qualitative lesson of this experiment is therefore twofold. On the one hand, the nominal single-target problem appears to favor direct optimization, which sets a strong baseline and usefully prevents overstating the value of RL in the easiest regime. On the other hand, the nominal TRPO result still shows that the policy-based approach is able to discover a fast analog solution in the relevant region of control space, with fidelity above \(99\%\) and a runtime substantially below the gate-synthesis estimate. This makes the next comparison especially meaningful: once stochastic training and robustness metrics are introduced, the question is no longer whether RL can produce a workable nominal pulse, but whether it can recover an advantage precisely along the dimensions for which the UFO framework was originally designed, namely robustness under control noise and improved stability of the learned control strategy.

\subsection{Comparative discussion}
Taken together, the single-target nominal experiments provide a fairly clear picture of the relative roles of direct pulse optimization and policy-based reinforcement learning in the present setting. On the representative target \(N(2.2,2.2,\pi/2)\), the Adam baseline yields the best overall nominal solution, with
\begin{equation}
C_{\mathrm{Adam}}^{\star}=0.1256,\qquad
F_{\mathrm{Adam}}=0.999385,\qquad
L_{\mathrm{Adam}}=1.98\times 10^{-4},\qquad
T_{\mathrm{Adam}}=70\,\mathrm{ns},
\end{equation}
while the best nominal TRPO controller achieves
\begin{equation}
C_{\mathrm{TRPO}}^{\star}=0.3676,\qquad
F_{\mathrm{TRPO}}=0.99155,\qquad
L_{\mathrm{TRPO}}=4.59\times 10^{-4},\qquad
T_{\mathrm{TRPO}}=102\,\mathrm{ns}.
\end{equation}
Therefore, on this fixed deterministic target, Adam outperforms nominal TRPO on all four primary metrics: total objective value, gate fidelity, leakage suppression, and runtime. The difference is not marginal. In particular, the fidelity gap is nearly one percentage point, the leakage of the RL solution is more than twice as large, and the learned TRPO pulse is approximately \(46\%\) longer than the best Adam pulse.

This result is scientifically useful, because it identifies quite sharply what can and cannot be claimed about the RL framework at the nominal level. If the question is whether a policy-gradient agent can discover a valid analog controller for the target gate, then the answer is clearly yes: the nominal TRPO run still reaches a fidelity above \(99\%\), keeps leakage below \(5\times10^{-4}\), and produces a gate duration of only \(102\,\mathrm{ns}\), which is substantially shorter than the approximate \(215\,\mathrm{ns}\) synthesized-gate reference reported in Ref.~\cite{Niu2019}. In that sense, the RL formulation is already \emph{competitive} with respect to the broad hardware-efficiency motivation of the UFO framework, since it finds a fast analog solution well inside the physically relevant regime. However, if the question is whether RL is already the best optimization method for the \emph{single-target nominal problem itself}, the answer from the present data is no: under the present settings, direct gradient-based pulse search remains clearly superior.

This hierarchy is also qualitatively consistent with the broader quantum-control literature. For deterministic fixed-target problems with a differentiable simulator and a fully specified cost function, direct optimization over the pulse parameters is often extremely effective, especially when the task is to extract the best possible nominal solution for one target gate \cite{Khaneja2005,deFouquieres2011,Machnes2018}. By contrast, the main conceptual appeal of the UFO framework is not simply that it replaces a strong optimizer by a weaker one, but that it embeds leakage, runtime, and eventually stochastic robustness into a unified sequential-learning formulation \cite{Niu2019}. From this perspective, the present nominal comparison is best interpreted as a control experiment: it shows that RL should not be justified by claiming a trivial nominal advantage where none is observed, and it motivates shifting the emphasis toward the regimes in which policy-based learning is actually expected to help, namely noisy-environment training, robustness under perturbations, and possibly transfer across related targets.

A further methodological point should also be stated explicitly. The present nominal comparison is deliberately fair in the sense that both methods use the same physical model, timestep, target gate, and UFO weights, but it is not perfectly symmetric in optimization protocol. In particular, the TRPO nominal run employed a looser stopping threshold than the Adam sweep, which likely favors earlier acceptance of a merely satisfactory policy rather than continued refinement toward the sharpest nominal optimum. This caveat should not be used to erase the observed gap, which remains substantial, but it does matter for interpretation: the current result shows that nominal TRPO is \emph{feasible and promising}, not that it has already been pushed to the same degree of local optimality as the direct Adam baseline.

The most balanced conclusion is therefore the following. In the nominal single-target regime, the Adam baseline is the more effective optimizer and provides the stronger reference pulse. Nevertheless, the TRPO controller still demonstrates that the RL-based UFO formulation can discover fast, high-fidelity analog controls that are already hardware-efficient relative to standard gate synthesis. The comparative value of RL, however, does not emerge most strongly in this deterministic benchmark. Its real test is whether the policy-based formulation becomes advantageous once stochastic training, robustness metrics, and more transferable control structure are taken into account. This is precisely the question addressed in the following sections.

\section{Robustness under stochastic control noise}
\label{sec:robustness_under_stochastic_control_noise}
\subsection{Noise model}
\label{subsec:noise_model}
To assess robustness in a physically meaningful and methodologically consistent way, all controllers considered in this section are evaluated under the same stochastic control-error model, following the general logic of the original UFO framework \cite{Niu2019}. At the discrete time-step level, the nominal control trajectory is not implemented exactly, but is perturbed by additive Gaussian amplitude fluctuations. In the notation of the control vector introduced in Eq.~\eqref{eq:validation_control_vector}, this means that the realized control at time step \(t_k\) takes the schematic form
\begin{equation}
\bm{u}^{(\mathrm{noisy})}(t_k)
=
\bm{u}(t_k)+\bm{\xi}(t_k),
\label{eq:robustness_noisy_control_vector}
\end{equation}
where the noise term \(\bm{\xi}(t_k)\) is sampled independently at each time step from a zero-mean Gaussian distribution. In line with both the original paper and the present implementation, these fluctuations are applied not only to the directly actuated control amplitudes, namely the coupling pulse \(g(t)\), the detuning amplitudes \(\delta_j(t)\), and the microwave amplitudes \(f_j(t)\), but also to the anharmonicity-related parameter \(\eta\), which enters the Hamiltonian as part of the multilevel device model \cite{Niu2019}. This point is important: the noise model is not restricted to imperfect pulse delivery alone, but also probes sensitivity to uncertainty in a physically relevant Hamiltonian parameter.

At the Hamiltonian level, each noisy rollout is therefore generated by replacing the nominal control-dependent evolution with a perturbed one-step propagator,
\begin{equation}
U_{k+1}
=
\exp\!\left[
-i\Delta t\,
\bigl(
\hat{H}(t_k)+\delta \hat{H}(t_k)
\bigr)
\right]
U_k,
\label{eq:robustness_noisy_propagator}
\end{equation}
where \(\delta \hat{H}(t_k)\) collects the Gaussian perturbations induced by the noisy realization of the control parameters and of \(\eta\). The overall noise strength is parametrized by a standard deviation \(\sigma_{\mathrm{noise}}\), which is varied systematically in the robustness study below. In this way, robustness is not inferred from a single perturbed trajectory, but from a family of stochastic control channels of increasing severity.

A further methodological distinction must be made between \emph{training noise} and \emph{evaluation noise}. For the Adam baseline and the nominal TRPO controller, optimization is performed in the deterministic environment, and robustness is assessed only \emph{a posteriori} through noisy Monte Carlo rollouts. By contrast, for the noise-optimized TRPO agent, the same Gaussian perturbation model is already activated during training, so that the policy is learned in a stochastic environment rather than in a nominal one. This distinction is central for the interpretation of the results: if the noise-optimized TRPO controller later exhibits both higher average fidelity and lower fidelity variance under noisy evaluation, then the improvement cannot be attributed to reinforcement learning in general, but specifically to the fact that robustness has been learned from exposure to the stochastic environment itself \cite{Niu2019}.

The noisy performance of a given controller is then characterized statistically by repeated sampling. For each value of \(\sigma_{\mathrm{noise}}\), a fixed optimized control plan is executed over many independent noisy realizations, and the resulting empirical distribution of final fidelities is used to estimate both the average fidelity and the fidelity variance. In other words, the robustness metrics reported in the next paragraphs are not single-shot quantities, but Monte Carlo estimates over an ensemble of noisy trajectories,
\begin{equation}
\overline{F}(\sigma_{\mathrm{noise}})
=
\mathbb{E}_{\delta\bm{u}\sim\mathcal{N}(0,\sigma_{\mathrm{noise}})}
\left[
F\bigl(U(\bm{u}+\delta\bm{u})\bigr)
\right],
\label{eq:robustness_average_fidelity_section}
\end{equation}
and
\begin{equation}
\mathrm{Var}(F)(\sigma_{\mathrm{noise}})
=
\mathbb{E}_{\delta\bm{u}}
\left[
\bigl(F-\overline{F}\bigr)^2
\right].
\label{eq:robustness_fidelity_variance_section}
\end{equation}
This evaluation protocol is particularly well suited to the present thesis, because it directly probes the central claim of the UFO framework: a useful control strategy should not only achieve high nominal fidelity, but should remain accurate and stable under realistic stochastic perturbations of the multilevel superconducting-qubit dynamics.


\subsection{Average fidelity versus noise strength.}
The main robustness trend is summarized in Panel~A of Fig.~\ref{fig:robustness_noise_comparison}, where the average fidelity is reported as a function of the Gaussian noise standard deviation \(\sigma_{\mathrm{noise}}\) for the three controllers considered in this study: the Adam baseline, the nominal TRPO controller, and the noise-optimized TRPO controller. For all methods, the average fidelity decreases monotonically as the noise strength is increased from the near-nominal regime up to the strongest perturbations considered in the sweep, which is the expected qualitative behavior for a control protocol subjected to progressively larger stochastic distortions. At very small \(\sigma_{\mathrm{noise}}\), all three curves start in the high-fidelity regime, consistently with the corresponding deterministic evaluations: the Adam plan begins from nominal fidelity \(0.99939\), the noise-optimized TRPO plan from \(0.99876\), and the nominal TRPO plan from \(0.99415\). 
% :contentReference[oaicite:0]{index=0} :contentReference[oaicite:1]{index=1} :contentReference[oaicite:2]{index=2}

As the noise amplitude grows, however, the three methods separate clearly. The Adam baseline remains the best-performing controller over the entire range shown in the figure, preserving the largest average fidelity at every noise level. This is a noteworthy result, because it shows that, for this specific single-target benchmark, the direct pulse optimizer does not only dominate in the deterministic setting discussed in Section~\ref{subsec:single_target_control_optimization}, but also retains the strongest mean performance under stochastic perturbations. Nevertheless, the comparison between the two RL curves is the more important result for the logic of the UFO framework. The noise-optimized TRPO controller consistently outperforms the nominally trained TRPO controller across essentially the whole sweep, and the gap between the two becomes progressively more visible as \(\sigma_{\mathrm{noise}}\) increases. In other words, although noisy-environment RL does not overtake Adam on this target, it does produce a controller whose mean noisy fidelity is systematically better than that of the RL agent trained only in the nominal environment.

This ranking is exactly the one that matters for answering the central question of this subsection: \emph{does noisy training genuinely improve robustness?} At the level of average fidelity, the answer is yes. Since the two RL controllers share the same general policy-based optimization framework and differ primarily in whether stochastic perturbations are present during training, their separation under noisy evaluation cannot be attributed merely to the use of reinforcement learning in general. Rather, it indicates that exposure to the stochastic environment during optimization leads the agent to discover control policies whose performance degrades more slowly under Gaussian perturbations. This conclusion is fully aligned with the intended logic of the original UFO paper, where the noisy training environment is introduced precisely in order to optimize expected performance under control uncertainty rather than nominal fidelity alone \cite{Niu2019}. At the same time, the fact that Adam still traces the highest curve reminds us that, in the present single-target study, robustness gains from noisy RL should be interpreted as \emph{relative to nominal RL}, not yet as an unconditional superiority over all alternative optimization strategies.

\subsection{Fidelity variance versus noise strength.}
A complementary view is provided by Panel~B of Fig.~\ref{fig:robustness_noise_comparison}, which reports the fidelity variance as a function of the same Gaussian noise amplitude \(\sigma_{\mathrm{noise}}\). Whereas the average fidelity quantifies the \emph{mean} degradation of a control protocol under stochastic perturbations, the variance probes its \emph{shot-to-shot stability}: a controller with smaller \(\mathrm{Var}(F)\) is less sensitive to the specific noise realization and therefore more reproducible in practice. This distinction is essential in the present context, because a method may retain an acceptable average fidelity while still being experimentally fragile if its performance fluctuates strongly across noisy rollouts. The use of fidelity variance as a second robustness diagnostic is therefore not incidental, but directly follows the logic of the original UFO framework, which evaluates robustness not only through mean performance but also through the stability of that performance under repeated stochastic perturbations \cite{Niu2019}. 
% :contentReference[oaicite:0]{index=0}

The qualitative trend is clear. For all three controllers, the fidelity variance is essentially negligible in the weak-noise regime and then increases progressively, with a markedly faster growth once the perturbation strength enters the intermediate-to-strong regime. This behavior is physically reasonable: when \(\sigma_{\mathrm{noise}}\) is very small, different noisy realizations remain close to the nominal control trajectory and therefore produce almost indistinguishable fidelities; as the fluctuations become larger, however, the controller begins to sample increasingly different effective Hamiltonian trajectories, and the output fidelity becomes more dispersed from one rollout to the next. In this sense, Panel~B provides a direct measure of sensitivity of the learned or optimized control plan to stochastic deformation of the pulse sequence.

The ordering of the three methods mirrors, but sharpens, the conclusion already suggested by the average-fidelity curves. The Adam baseline exhibits the lowest fidelity variance over the full sweep, indicating that it is also the most stable controller under repeated noisy realizations on this target. Among the two RL-based methods, however, the noise-optimized TRPO controller remains systematically below the nominally trained TRPO controller across essentially the whole range of \(\sigma_{\mathrm{noise}}\). Thus, even though noisy-environment RL does not achieve the best absolute robustness in this single-target comparison, it does produce a clear reduction in shot-to-shot variability relative to nominal RL. This is an important result, because fidelity variance is a stricter robustness criterion than average fidelity alone: it shows not only that the noise-optimized controller performs better on average, but also that its performance is less erratic under the same stochastic perturbation model.

Another useful feature of Panel~B is that the separation between the two RL curves becomes more pronounced as the noise level increases. At low noise, both variances remain very small, so the practical difference between nominal and noisy training is limited. At larger \(\sigma_{\mathrm{noise}}\), by contrast, the variance of the nominal TRPO controller grows more rapidly, while the noise-optimized controller retains a visibly lower dispersion. This suggests that stochastic training does not merely shift the mean fidelity upward by a small amount, but changes the \emph{sensitivity profile} of the learned controller itself. In other words, the policy trained in the noisy environment appears to be less brittle with respect to random perturbations of the control amplitudes and of the Hamiltonian parameter \(\eta\), which is precisely the type of robustness the stochastic-training strategy was designed to encourage.

\subsection{Interpretation.}
Taken together, Panels~A and~B of Fig.~\ref{fig:robustness_noise_comparison} support the central mechanism underlying the UFO framework, but in a more nuanced form than a simple method-ranking statement. The key observation is not merely that the three controllers degrade differently under stochastic perturbations, but that the \emph{relative ordering between the two RL-based controllers is stable across both robustness diagnostics}. The noise-optimized TRPO controller achieves a systematically higher average fidelity than the nominally trained TRPO controller, while at the same time maintaining a systematically lower fidelity variance over essentially the entire noise sweep. This joint improvement is the crucial result of the subsection. If stochastic training improved only the mean fidelity but produced larger shot-to-shot fluctuations, or reduced the variance at the price of a substantially worse mean fidelity, the interpretation would remain ambiguous. Instead, the noisy-environment controller improves both quantities simultaneously. This strongly suggests that the robustness gain is not a generic side effect of using reinforcement learning, but a property that is learned through exposure to the stochastic environment during training \cite{Niu2019}.

This conclusion is particularly important in light of the comparison with the nominal TRPO controller. Since the two RL agents share the same general policy-based architecture, the same physical model, and the same class of cost function, their different behavior under noisy evaluation is naturally traced back to the one ingredient that distinguishes them most directly: whether the policy was optimized in a deterministic environment or in a stochastic one. In this sense, the present results validate the main conceptual thesis of the paper at the level most relevant for this chapter: training under control noise changes the learned policy in a way that makes it less sensitive to subsequent random perturbations. That is, the observed robustness is not merely \emph{incidental}, but \emph{acquired}. This is exactly the interpretation that gives physical meaning to the noisy-environment RL strategy, and it is the aspect of the UFO framework that is most clearly reproduced by the present implementation.

At the same time, the same figures also show that this learned robustness should not be overinterpreted. On the representative target considered here, the Adam baseline remains superior in absolute terms, both in average fidelity and in fidelity variance, across the full range of tested noise amplitudes. Therefore, the present robustness study does not support the strongest possible claim that noisy-environment TRPO dominates all competing optimization strategies on this single-target benchmark. Rather, it supports a more precise and scientifically more defensible conclusion: within the RL family, stochastic training produces a controller that is genuinely more robust than nominal training, while among the three controllers considered here the direct pulse optimizer still provides the strongest absolute performance on this target. This distinction is important for the overall logic of the thesis. It prevents the empirical analysis from overstating the success of RL in the easiest single-target regime, while at the same time confirming the specific paper-level hypothesis that robustness can be intentionally learned by training in a stochastic environment.

From the viewpoint of the broader thesis, this subsection therefore plays a pivotal role. The nominal single-target study of Section~\ref{subsec:single_target_control_optimization} had shown that direct optimization is already extremely strong when the problem is deterministic and fixed. The present robustness study adds the missing piece: once stochastic perturbations are treated as part of the optimization problem rather than as an afterthought, the noisy-environment RL formulation begins to exhibit its intended advantage, namely a statistically more stable and less noise-sensitive control policy than the one obtained by nominal RL alone. For this reason, even though the absolute best controller in this benchmark remains the Adam baseline, the results of Fig.~\ref{fig:robustness_noise_comparison} should be regarded as a substantive validation of the most characteristic claim of the UFO framework.


\section{Runtime analysis on the gate family \(N(\alpha,\alpha,\pi/2)\)}
\paragraph{Runtime sweep protocol.}
To investigate the hardware efficiency of analog control beyond the single-target setting, I next considered the one-parameter subfamily
\begin{equation}
N(\alpha,\alpha,\pi/2),
\qquad
\alpha \in [0,\pi],
\label{eq:runtime_subfamily_empirical}
\end{equation}
which is the most relevant runtime benchmark discussed in the original UFO paper \cite{Niu2019}. In the present implementation, the sweep was performed by fixing \(\gamma=\pi/2\) and varying \(\alpha\) from \(0\) to approximately \(\pi\) with step size \(0.157\), corresponding to a discretization of the target family into a sequence of nearby gates. For each value of \(\alpha\), a noise-optimized TRPO controller was trained and then evaluated deterministically to extract the runtime of the best control plan found for that target. As in the rest of the chapter, the physical model, timestep, and network architecture were kept fixed, with \(\Delta t = 2\,\mathrm{ns}\), maximum admissible gate duration \(180\,\mathrm{ns}\), and the same \(64\text{--}32\text{--}32\) policy/value architecture used throughout the study. The training environment remained stochastic, with Gaussian control noise of standard deviation \(1\,\mathrm{MHz}\), so that the runtime analysis was not performed for purely nominal policies, but for controllers optimized under the same noisy-environment logic that underlies the UFO framework \cite{Niu2019}. :contentReference[oaicite:0]{index=0} :contentReference[oaicite:1]{index=1}

A central methodological ingredient of this runtime study is the use of \emph{curriculum-style transfer} across nearby targets. Rather than training a fresh agent independently for each \(\alpha\), the best controller learned for one value of \(\alpha\) was carried forward through a curriculum checkpoint and used to initialize training for the next point in the sweep. In this way, the agent was not repeatedly asked to rediscover the entire control structure from scratch, but instead exploited the continuity of the gate family in parameter space. This protocol is directly inspired by the strategy described in the original paper, where the same RL agent is advanced through neighboring target gates in order to reduce the overall optimization burden and to exploit transfer between nearby tasks \cite{Niu2019}. In the present codebase, this idea is implemented explicitly through the saving of a curriculum checkpoint after each completed \(\alpha\)-phase and its reuse as the initialization for the next one. Training for a given target was then stopped either when a sufficiently low control cost was reached or when the maximum per-target iteration budget was exhausted, thereby producing a practical runtime sweep with bounded computational cost rather than an independent full optimization at every point. :contentReference[oaicite:2]{index=2}

The quantity of interest in this section is the final analog gate duration extracted from the best deterministic rollout at each \(\alpha\). This runtime is then compared against the standard optimal gate-synthesis reference for the same gate family. Following Ref.~\cite{Niu2019}, the discrete synthesis benchmark is constructed from the minimum-depth decomposition of \(N(\alpha,\alpha,\gamma)\) into three two-qubit entangling gates and five single-qubit rotations, with state-of-the-art experimental gate times \(T_{1q}=20\,\mathrm{ns}\) and \(T_{\mathrm{CNOT}}=45\,\mathrm{ns}\), yielding a total synthesized runtime of approximately
\begin{equation}
T_{\mathrm{syn}} \approx 215\,\mathrm{ns}.
\label{eq:runtime_synthesis_reference_empirical}
\end{equation}
This reference plays the role of a fixed hardware-efficiency baseline throughout the subsection. The central question is therefore not whether the learned controls are simply short in absolute terms, but whether, along the family \(N(\alpha,\alpha,\pi/2)\), they can remain shorter than the best known standard discrete implementation. This is exactly the runtime question posed by the original paper, and it provides the natural benchmark for the empirical analysis that follows.
